# 技巧与最佳实践

一份能迅速见效的实用技巧合集，助您即刻提升使用 Hermes 智能体的效率。每个章节针对不同方面 — 浏览标题，直接跳转到相关部分。

---

## 获取最佳结果

### 明确您的需求

模糊的提示会得到模糊的结果。不要说“修复代码”，而是说“修复 `api/handlers.py` 第 47 行的 TypeError — `process_request()` 函数从 `parse_body()` 接收到了 `None`”。您提供的上下文越多，所需的迭代次数就越少。

### 提前提供上下文

在请求的开头就提供相关细节：文件路径、错误消息、预期行为。一条精心设计的消息胜过三轮澄清。直接粘贴错误回溯信息 — 智能体可以解析它们。

### 对重复指令使用上下文文件

如果您发现自己重复相同的指令（“使用制表符而非空格”、“我们使用 pytest”、“API 位于 `/api/v2`”），请将它们放入 `AGENTS.md` 文件。智能体会在每次会话中自动读取 — 设置完成后无需额外操作。

### 让智能体使用其工具

不要试图手把手指导每一步。说“找到并修复失败的测试”，而不是“打开 `tests/test_foo.py`，查看第 42 行，然后...”。智能体拥有文件搜索、终端访问和代码执行能力 — 让它自行探索和迭代。

### 对复杂工作流使用技能

在撰写冗长提示以解释如何完成某事之前，请检查是否已有适用的技能。输入 `/skills` 浏览可用技能，或直接调用一个，例如 `/axolotl` 或 `/github-pr-workflow`。

## CLI 高级用户技巧

### 多行输入

按 **Alt+Enter**、**Ctrl+J** 或 **Shift+Enter** 插入新行而不发送消息。`Shift+Enter` 仅在终端将其作为独立按键发送时有效（默认在 Kitty / foot / WezTerm / Ghostty 中生效；在 iTerm2 / Alacritty / VS Code 终端中启用 Kitty 键盘协议后也可生效）。另外两个快捷键在所有终端中均有效。

### 粘贴检测

CLI 可自动检测多行粘贴。只需直接粘贴代码块或错误回溯信息——它不会将每一行作为单独的消息发送。粘贴的内容会被缓冲并作为一条消息发送。

### 中断与重定向

按 **Ctrl+C** 一次可在智能体响应过程中中断它。然后您可以输入新消息来重定向它。在2秒内双击 Ctrl+C 可强制退出。当智能体开始偏离正轨时，这非常有用。

### 使用 `-c` 恢复会话

忘记了上一个会话的内容？运行 `hermes -c` 即可恢复到您离开时的状态，完整会话历史记录将被恢复。您也可以通过标题恢复：`hermes -r "我的研究项目"`。

### 剪贴板图像粘贴

按 **Ctrl+V** 可将剪贴板中的图像直接粘贴到聊天中。智能体使用视觉功能来分析截图、图表、错误弹窗或 UI 模型——无需先保存为文件。

### 斜杠命令自动补全

输入 `/` 并按 **Tab** 可查看所有可用命令。这包括内置命令（`/compress`、`/model`、`/title`）和每个已安装的技能。您无需记住任何内容——Tab 补全功能会帮您搞定。

:::tip
使用 `/verbose` 可循环切换工具输出显示模式：**关闭 → 新增 → 全部 → 详细**。“全部”模式非常适合观察智能体的操作；“关闭”模式对于简单的问答最为简洁。
:::

## 上下文文件

### AGENTS.md：您项目的大脑

在项目根目录创建一个 `AGENTS.md` 文件，包含架构决策、编码规范和项目特定说明。该文件会自动注入到每个会话中，因此智能体始终了解您项目的规则。

```markdown
# 项目上下文
- 这是一个使用 SQLAlchemy ORM 的 FastAPI 后端
- 数据库操作始终使用 async/await
- 测试放在 tests/ 目录下，并使用 pytest-asyncio
- 永远不要提交 .env 文件
```

### SOUL.md：自定义个性

希望 Hermes 拥有一个稳定的默认声音？编辑 `~/.hermes/SOUL.md`（如果您使用自定义的 Hermes 主目录，则是 `$HERMES_HOME/SOUL.md`）。Hermes 现在会自动生成一个起始 SOUL 文件，并使用该全局文件作为实例范围的个性来源。

完整演练请参阅 [与 Hermes 一起使用 SOUL.md](/guides/use-soul-with-hermes)。

```markdown
# 灵魂
你是一名高级后端工程师。简洁直接。
除非被要求，否则跳过解释。偏好单行解决方案而非冗长方案。
始终考虑错误处理和边缘情况。
```

使用 `SOUL.md` 保存持久个性。使用 `AGENTS.md` 保存项目特定指令。

### .cursorrules 兼容性

已有 `.cursorrules` 或 `.cursor/rules/*.mdc` 文件？Hermes 也会读取它们。无需重复编写编码规范——它们会从工作目录自动加载。

### 发现机制

Hermes 在会话开始时会从当前工作目录加载顶层的 `AGENTS.md`。子目录下的 `AGENTS.md` 文件在工具调用过程中会被懒加载发现（通过 `subdirectory_hints.py`）并注入到工具结果中——它们不会预先加载到系统提示中。

:::tip
保持上下文文件精简聚焦。每个字符都会计入您的令牌预算，因为它们会被注入到每条消息中。
:::

## 记忆与技能

### 记忆与技能：各自存放什么

**记忆**用于存储事实：您的环境、偏好、项目位置以及智能体已了解的关于您的信息。**技能**用于存储流程：多步骤工作流、特定工具的指令和可重用方案。记忆用于“是什么”，技能用于“怎么做”。

### 何时创建技能

如果您发现一个需要 5 个以上步骤且会重复执行的任务，可以让智能体为此创建一个技能。说“将你刚才做的事情保存为一个名为 `deploy-staging` 的技能”。下次只需输入 `/deploy-staging`，智能体就会加载完整流程。

### 管理记忆容量

记忆容量是有限制的（MEMORY.md 约 2,200 个字符，USER.md 约 1,375 个字符）。当它填满时，智能体会合并条目。您可以说“清理你的记忆”或“替换掉旧的 Python 3.9 说明——我们现在用的是 3.12”来帮助管理。

### 让智能体记住

在一次富有成效的会话后，说“为下次记住这个”，智能体会保存关键要点。您也可以更具体：“保存到记忆中，我们的 CI 使用 GitHub Actions 并配合 `deploy.yml` 工作流。”

:::warning
记忆是冻结的快照——会话期间进行的更改在下一个会话开始之前不会出现在系统提示中。智能体会立即写入磁盘，但提示缓存不会在会话中途失效。
:::

## 性能与成本

### 不要破坏提示缓存

大多数 LLM 提供商都会缓存系统提示的前缀。如果您保持系统提示稳定（相同的上下文文件、相同的记忆），会话中后续消息的**缓存命中率**会显著提高，成本也更低。避免在会话中途更换模型或系统提示。

### 在达到限制前使用 /compress

长时间的会话会累积令牌。当您注意到响应变慢或被截断时，运行 `/compress`。这会总结对话历史，保留关键上下文，同时大幅减少令牌数量。使用 `/usage` 查看您的使用情况。

### 委托以实现并行工作

需要同时研究三个主题？让智能体使用 `delegate_task` 进行并行子任务。每个子智能体独立运行并有自己的上下文，只有最终摘要会返回——这大幅减少了您主对话的令牌使用量。

### 使用 execute_code 进行批量操作

与其一次运行一个终端命令，不如让智能体编写一个脚本来一次性完成所有操作。“编写一个 Python 脚本，将所有 `.jpeg` 文件重命名为 `.jpg` 并运行它”比逐个重命名文件更便宜、更快捷。

### 选择合适的模型

使用 `/model` 在会话中途切换模型。对于复杂的推理和架构决策，使用前沿模型（Claude Sonnet/Opus、GPT-4o）。对于简单的任务，如格式化、重命名或生成样板代码，切换到更快的模型。

:::tip
定期运行 `/usage` 查看您的令牌消耗。运行 `/insights` 可查看过去 30 天的更广泛使用模式。
:::

## 消息传递技巧

### 设置主频道

在您首选的 Telegram 或 Discord 聊天中使用 `/sethome` 将其指定为主频道。定时任务结果和计划任务输出将发送到这里。如果没有设置，智能体将没有地方发送主动消息。

### 使用 /title 组织会话

使用 `/title auth-refactor` 或 `/title research-llm-quantization` 为您的会话命名。命名会话可以通过 `hermes sessions list` 轻松找到，并通过 `hermes -r "auth-refactor"` 恢复。未命名会话会堆积起来，变得无法区分。

### DM 配对以实现团队访问

无需手动收集用户 ID 来建立允许列表，启用 DM 配对功能。当团队成员向机器人发送私信时，他们会收到一个一次性配对码。您可以通过 `hermes pairing approve telegram XKGH5N7P` 来批准——简单又安全。

### 工具进度显示模式

使用 `/verbose` 控制您看到的工具活动量。在消息传递平台上，少即是多——保持为“新增”以仅查看新的工具调用。在 CLI 中，“全部”能让您满意地实时查看智能体的所有操作。

:::tip
在消息传递平台上，会话会在闲置时间（默认：24 小时）后或每天凌晨 4 点自动重置。如果需要更长的会话，可以在 `~/.hermes/config.yaml` 中按平台调整。
:::

## 安全性

### 对不可信代码使用 Docker

在处理不可信仓库或运行不熟悉的代码时，请使用 Docker 或 Daytona 作为您的终端后端。在您的 `.env` 文件中设置 `TERMINAL_BACKEND=docker`。容器内的破坏性命令不会损害您的主机系统。

```bash
# 在您的 .env 文件中：
TERMINAL_BACKEND=docker
TERMINAL_DOCKER_IMAGE=hermes-sandbox:latest
```

### 避免 Windows 编码陷阱

在 Windows 上，某些默认编码（如 `cp125x`）无法表示所有 Unicode 字符，这可能在测试或脚本中写入文件时导致 `UnicodeEncodeError`。

- 尽量使用显式的 UTF-8 编码打开文件：

```python
with open("results.txt", "w", encoding="utf-8") as f:
    f.write("✓ All good\n")
```

- 在 PowerShell 中，您也可以将当前会话切换为 UTF-8，用于控制台和原生命令输出：

```powershell
$OutputEncoding = [Console]::OutputEncoding = [Text.UTF8Encoding]::new($false)
```

这使 PowerShell 和子进程保持在 UTF-8 编码下，并有助于避免仅 Windows 环境下的故障。

### 在选择“始终”前进行审查

当智能体触发危险命令审批（`rm -rf`、`DROP TABLE` 等）时，您会看到四个选项：**一次**、**本次会话**、**始终**、**拒绝**。在选择“始终”之前请三思——它会永久将该模式加入允许列表。在您放心之前，先选择“本次会话”。

### 命令审批是您的安全网

Hermes 在执行每个命令前，都会对照一个精心策划的危险模式列表进行检查。这包括递归删除、SQL 删除、将 curl 管道传输到 shell 等。不要在生产环境中禁用此功能——它存在是有充分理由的。

:::warning
在容器后端（Docker、Singularity、Modal、 Daytona）中运行时，危险命令检查会被**跳过**，因为容器本身就是安全边界。请确保您的容器镜像已妥善锁定。
:::

### 对消息传递机器人使用允许列表

永远不要在具有终端访问权限的机器人上设置 `GATEWAY_ALLOW_ALL_USERS=true`。始终使用特定平台的允许列表（`TELEGRAM_ALLOWED_USERS`、`DISCORD_ALLOWED_USERS`）或 DM 配对来控制谁可以与您的智能体交互。

```bash
# 推荐：为每个平台设置明确的允许列表
TELEGRAM_ALLOWED_USERS=123456789,987654321
DISCORD_ALLOWED_USERS=123456789012345678

# 或使用跨平台允许列表
GATEWAY_ALLOWED_USERS=123456789,987654321
```
