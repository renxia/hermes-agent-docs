# 技巧与最佳实践

这是一份快速见效的实用技巧合集，能让你立即更高效地使用 Hermes Agent。每个部分针对不同的方面 —— 浏览标题并跳转到相关内容即可。

---

## 获得最佳结果

### 明确说明你的需求

模糊的提示会产生模糊的结果。与其说“修复代码”，不如说“修复 `api/handlers.py` 第 47 行的 TypeError —— `process_request()` 函数从 `parse_body()` 接收到 `None`。”你提供的上下文越多，所需的迭代就越少。

### 预先提供上下文

在请求开头就提供相关细节：文件路径、错误信息、预期行为。一条精心编写的消息胜过三轮澄清。直接粘贴错误堆栈跟踪 —— 智能体可以解析它们。

### 对重复性指令使用上下文文件

如果你发现自己反复使用相同的指令（“使用制表符而不是空格”、“我们使用 pytest”、“API 地址是 `/api/v2`”），请将它们放入 `AGENTS.md` 文件。智能体会在每次会话中自动读取该文件 —— 设置完成后零额外操作。

### 让智能体使用其工具

不要试图手把手指导每一步。与其说“打开 `tests/test_foo.py`，查看第 42 行，然后……”，不如说“查找并修复失败的测试”。智能体具备文件搜索、终端访问和代码执行能力 —— 让它自行探索和迭代。

### 对复杂工作流使用技能

在编写冗长提示解释如何完成某项任务之前，先检查是否已有对应的技能。输入 `/skills` 浏览可用技能，或直接调用某个技能，例如 `/axolotl` 或 `/github-pr-workflow`。

## CLI 高级用户技巧

### 多行输入

按下 **Alt+Enter**、**Ctrl+J** 或 **Shift+Enter** 可在不发送消息的情况下插入换行符。`Shift+Enter` 仅在终端将其识别为独立按键时生效（默认情况下适用于 Kitty / foot / WezTerm / Ghostty；iTerm2 / Alacritty / VS Code 终端需在启用 Kitty 键盘协议后生效）。其余两个快捷键在所有终端中均可使用。

### 粘贴检测

CLI 会自动检测多行粘贴内容。直接粘贴代码块或错误堆栈跟踪即可——不会将每一行作为单独消息发送。粘贴内容会被缓冲，并作为一条消息发送。

### 中断与重定向

按一次 **Ctrl+C** 可中断智能体正在进行的回复。随后您可以输入新消息来重定向其任务。在 2 秒内双击 Ctrl+C 可强制退出。当智能体开始走错方向时，此功能非常有用。

### 使用 `-c` 恢复会话

忘记上次会话中的某些内容？运行 `hermes -c` 即可从上次离开的位置完全恢复，包括完整的对话历史记录。您也可以通过标题恢复：`hermes -r "my research project"`。

### 剪贴板图片粘贴

按 **Ctrl+V** 可直接将剪贴板中的图片粘贴到聊天中。智能体利用视觉能力分析截图、图表、错误弹窗或 UI 原型——无需先保存为文件。

### 斜杠命令自动补全

输入 `/` 并按 **Tab** 键可查看所有可用命令。包括内置命令（`/compress`、`/model`、`/title`）以及每个已安装的技能。您无需记忆任何内容——Tab 自动补全会为您提供支持。

:::tip
使用 `/verbose` 可在工具输出显示模式之间循环切换：**关闭 → 新增 → 全部 → 详细**。“全部”模式非常适合观察智能体的操作；“关闭”模式则最适合简单的问答场景。
:::

## 上下文文件

### AGENTS.md：项目的“大脑”

在项目根目录创建 `AGENTS.md` 文件，用于记录架构决策、编码约定和项目特定说明。该文件会自动注入每次会话中，因此智能体始终了解您项目的规则。

```markdown
# 项目上下文
- 这是一个使用 SQLAlchemy ORM 的 FastAPI 后端
- 数据库操作始终使用 async/await
- 测试文件放在 tests/ 目录下，并使用 pytest-asyncio
- 切勿提交 .env 文件
```

### SOUL.md：自定义人格

希望 Hermes 拥有稳定的默认语气？请编辑 `~/.hermes/SOUL.md`（如果您使用自定义 Hermes 主目录，则为 `$HERMES_HOME/SOUL.md`）。Hermes 现在会自动生成一个初始 SOUL 文件，并将其作为全局实例级人格来源。

完整教程请参阅 [在 Hermes 中使用 SOUL.md](/docs/guides/use-soul-with-hermes)。

```markdown
# 人格
你是一名资深后端工程师。请简洁直接。
除非被要求，否则跳过解释。优先使用一行式解决方案而非冗长方案。
始终考虑错误处理和边缘情况。
```

使用 `SOUL.md` 定义持久化人格。使用 `AGENTS.md` 定义项目特定说明。

### .cursorrules 兼容性

已有 `.cursorrules` 或 `.cursor/rules/*.mdc` 文件？Hermes 也会读取这些文件。无需重复编写编码约定——它们会自动从工作目录加载。

### 发现机制

Hermes 在会话启动时会加载当前工作目录下的顶层 `AGENTS.md` 文件。子目录中的 `AGENTS.md` 文件会在工具调用期间惰性发现（通过 `subdirectory_hints.py`），并注入工具结果中——它们不会预先加载到系统提示中。

:::tip
保持上下文文件聚焦且简洁。由于它们会被注入每条消息中，每个字符都会计入您的 token 预算。
:::

## 记忆与技能

### 记忆 vs. 技能：各自用途

**记忆**用于存储事实：您的环境、偏好、项目位置以及智能体已了解到的关于您的信息。**技能**用于存储流程：多步骤工作流、特定工具说明和可复用的配方。记忆用于“是什么”，技能用于“怎么做”。

### 何时创建技能

如果发现某项任务需要 5 步以上且您会再次执行，请让智能体为其创建一个技能。例如：“将你刚才所做的保存为一个名为 `deploy-staging` 的技能”。下次只需输入 `/deploy-staging`，智能体就会加载完整的流程。

### 管理记忆容量

记忆 intentionally 有限（MEMORY.md 约 2,200 字符，USER.md 约 1,375 字符）。当记忆填满时，智能体会合并条目。您可以说“清理你的记忆”或“替换旧的 Python 3.9 备注——我们现在用的是 3.12”来协助管理。

### 让智能体记住

在富有成效的会话结束后，说“下次记住这些内容”，智能体就会保存关键要点。您也可以更具体：“将我们的 CI 使用 GitHub Actions 和 `deploy.yml` 工作流这一信息保存到记忆中。”

:::warning
记忆是冻结的快照——会话期间所做的更改不会出现在系统提示中，直到下次会话开始。智能体会立即写入磁盘，但会话中途不会使提示缓存失效。
:::

## 性能与成本

### 不要破坏提示缓存

大多数 LLM 提供商会缓存系统提示前缀。如果您保持系统提示稳定（相同的上下文文件、相同的记忆），则会话中的后续消息将获得显著更便宜的**缓存命中**。避免在会话中途更改模型或系统提示。

### 在达到限制前使用 /compress

长时间会话会累积 token。当您注意到响应变慢或被截断时，请运行 `/compress`。这会总结对话历史记录，在保留关键上下文的同时大幅减少 token 数量。使用 `/usage` 查看当前状态。

### 委派以并行工作

需要同时研究三个主题？请让智能体使用 `delegate_task` 并设置并行子任务。每个子智能体独立运行并拥有自己的上下文，只有最终摘要会返回——从而大幅降低主对话的 token 使用量。

### 使用 execute_code 进行批量操作

与其一次运行一条终端命令，不如让智能体编写一个一次性完成所有操作的脚本。“编写一个 Python 脚本，将所有 `.jpeg` 文件重命名为 `.jpg` 并运行它”比逐个重命名文件更便宜、更快。

### 选择合适的模型

使用 `/model` 在会话中途切换模型。对于复杂推理和架构决策，请使用前沿模型（Claude Sonnet/Opus、GPT-4o）。对于格式化、重命名或模板生成等简单任务，请切换到更快的模型。

:::tip
定期运行 `/usage` 查看您的 token 消耗情况。运行 `/insights` 可更全面地了解过去 30 天的使用模式。
:::

## 消息技巧

### 设置主频道

在您偏好的 Telegram 或 Discord 聊天中使用 `/sethome` 将其指定为主频道。Cron 作业结果和计划任务输出将发送到此频道。如果没有设置，智能体将无处发送主动消息。

### 使用 /title 组织会话

使用 `/title auth-refactor` 或 `/title research-llm-quantization` 为您的会话命名。命名后的会话可通过 `hermes sessions list` 轻松查找，并通过 `hermes -r "auth-refactor"` 恢复。未命名的会话会堆积起来，变得难以区分。

### 团队访问的 DM 配对

无需手动收集用户 ID 以加入白名单，启用 DM 配对即可。当队友向机器人发送私信时，他们会获得一次性配对码。您可通过 `hermes pairing approve telegram XKGH5N7P` 批准——简单且安全。

### 工具进度显示模式

使用 `/verbose` 控制您看到的工具活动量。在消息平台中，通常“少即是多”——保持“新增”模式，仅查看新的工具调用。在 CLI 中，“全部”模式可让您满意地实时查看智能体所做的所有操作。

:::tip
在消息平台中，会话在空闲一段时间后（默认：24 小时）或每天凌晨 4 点自动重置。如需更长的会话，请在 `~/.hermes/config.yaml` 中按平台调整。
:::

## 安全

### 对不可信代码使用 Docker

在处理不可信仓库或运行陌生代码时，请使用 Docker 或 Daytona 作为终端后端。在 `.env` 中设置 `TERMINAL_BACKEND=docker`。容器内的破坏性命令无法损害您的主机系统。

```bash
# 在您的 .env 中：
TERMINAL_BACKEND=docker
TERMINAL_DOCKER_IMAGE=hermes-sandbox:latest
```

### 避免 Windows 编码陷阱

在 Windows 上，某些默认编码（如 `cp125x`）无法表示所有 Unicode 字符，这可能导致在测试或脚本中写入文件时出现 `UnicodeEncodeError`。

- 优先使用显式 UTF-8 编码打开文件：

```python
with open("results.txt", "w", encoding="utf-8") as f:
    f.write("✓ 一切正常\n")
```

- 在 PowerShell 中，您还可以将当前会话切换为 UTF-8，以用于控制台和原生命令输出：

```powershell
$OutputEncoding = [Console]::OutputEncoding = [Text.UTF8Encoding]::new($false)
```

这可使 PowerShell 和子进程保持 UTF-8 编码，并有助于避免仅限 Windows 的失败。

### 选择“始终”前请仔细审查

当智能体触发危险命令审批（`rm -rf`、`DROP TABLE` 等）时，您有四个选项：**一次**、**会话**、**始终**、**拒绝**。选择“始终”前请仔细考虑——它将永久将该模式加入白名单。建议先使用“会话”，直到您感到满意为止。

### 命令审批是您的安全网

Hermes 在执行前会检查每条命令是否匹配精心策划的危险模式列表。包括递归删除、SQL 删除、将 curl 输出通过管道传递给 shell 等。请勿在生产环境中禁用此功能——它的存在是有充分理由的。

:::warning
在容器后端（Docker、Singularity、Modal、Daytona）中运行时，**会跳过**危险命令检查，因为容器本身就是安全边界。请确保您的容器镜像已正确锁定。
:::

### 对消息机器人使用白名单

切勿在具有终端访问权限的机器人上设置 `GATEWAY_ALLOW_ALL_USERS=true`。始终使用平台特定白名单（`TELEGRAM_ALLOWED_USERS`、`DISCORD_ALLOWED_USERS`）或 DM 配对来控制谁可以与您的智能体交互。

```bash
# 推荐：按平台设置显式白名单
TELEGRAM_ALLOWED_USERS=123456789,987654321
DISCORD_ALLOWED_USERS=123456789012345678

# 或使用跨平台白名单
GATEWAY_ALLOWED_USERS=123456789,987654321
```
