# 教程：构建每日简报机器人

在本教程中，您将构建一个个人简报机器人，它每天早上唤醒，研究您关心的主题，总结发现结果，并将简洁的简报直接发送给您。

到本教程结束时，您将拥有一个完全自动化的工作流程，结合了**网络搜索**、**定时任务调度**、**委托执行**和**消息传递**——无需编写代码。

## 我们将构建什么

以下是流程：

1. **上午8:00** — 定时任务调度器触发您的任务
2. **Hermes启动**一个带有您提示语的新鲜智能体会话
3. **网络搜索**获取您主题的 latest 新闻
4. **摘要生成**将其提炼为清晰的简报格式
5. **发送简报**将简报发送到您的 Telegram 或 Discord

整个过程完全自动化运行。您只需在享用早晨咖啡时阅读简报即可。

## 先决条件

开始前，请确保您已具备以下条件：

- **安装了 Hermes Agent** — 参见 [安装指南](/docs/getting-started/installation)
- **Gateway 正在运行** — Gateway 守护进程处理定时任务执行：
  ```bash
  hermes gateway install   # 作为用户服务安装
  sudo hermes gateway install --system   # Linux 服务器：开机自启系统服务
  # 或者
  hermes gateway           # 在前台运行
  ```
- **Firecrawl API 密钥** — 设置环境变量 `FIRECRAWL_API_KEY` 以启用网络搜索
- **消息传递配置**（可选但推荐）— 使用 [Telegram](/docs/user-guide/messaging/telegram) 或 Discord 设置家庭频道

:::tip 没有消息传递？没问题
您仍可使用 `deliver: "local"` 跟随此教程。简报将保存到 `~/.hermes/cron/output/`，您可以随时查看。
:::

## 步骤 1：手动测试工作流

在自动化任何内容前，请先确保简报功能正常工作。开始一个聊天会话：

```bash
hermes
```

然后输入以下提示语：

```
搜索有关 AI 智能体和开源大语言模型的最新新闻。
用简洁的简报格式总结最重要的 3 条新闻，并附上链接。
```

Hermes 将进行网络搜索，浏览搜索结果，并生成如下内容：

```
☀️ 您的 AI 简报 — 2026年3月8日

1. Qwen 3 发布，参数达 235B
   阿里巴巴最新的开源模型在多个基准测试中与 GPT-4.5 表现相当，同时保持完全开源。
   → https://qwenlm.github.io/blog/qwen3/

2. LangChain 推出智能体协议标准
   一个新的智能体间通信开放标准在首周内被 15 个主流框架采用。
   → https://blog.langchain.dev/agent-protocol/

3. 欧盟 AI 法案开始对通用模型执行
   首个合规截止日期到来，开源模型在 1000 万参数阈值下获得豁免。
   → https://artificialintelligenceact.eu/updates/

---
3 条新闻 • 搜索来源：8 个 • 由 Hermes 智能体生成
```

如果这能正常工作，您就可以开始自动化了。

:::tip 迭代优化格式
尝试不同的提示语，直到获得您喜欢的输出。添加指令如“使用表情符号标题”或“每条摘要不超过两句话”。最终确定的内容将用于定时任务。
:::

## 步骤 2：创建定时任务

现在让我们安排它每天早上自动运行。您可以通过两种方式完成。

创建定时任务前，请确保 Hermes 已全局配置默认模型和提供商。如果您希望某个特定任务使用不同的值，请在创建时为它设置明确的每任务模型和提供商覆盖。

### 选项 A：自然语言（在聊天中）

只需告诉 Hermes 您想要什么：

```
每天早上 8 点，搜索有关 AI 智能体和开源大语言模型的最新新闻。
总结最重要的 3 条新闻，并附上链接。使用友好、专业的语气。发送到 Telegram。
```

Hermes 将使用统一的 `cronjob` 工具为您创建定时任务。

### 选项 B：CLI 斜杠命令

使用 `/cron` 命令以获得更多控制：

```
/cron add "0 8 * * *" "搜索有关 AI 智能体和开源大语言模型的最新新闻。
找到过去 24 小时内至少 5 篇最近的文章。
总结最重要的 3 条新闻，并以简洁的每日简报格式呈现。
每条新闻包括：清晰标题、两句话摘要和来源 URL。
使用友好、专业的语气。
使用表情符号项目符号格式化，并以总新闻数结尾。"
```

### 黄金法则：自包含提示语

:::warning 关键概念
定时任务在**全新的会话**中运行——不会记住之前的对话，也不会有任何关于您“之前设置的上下文”。您的提示语必须包含**智能体完成工作所需的一切**。
:::

**糟糕的提示语：**
```
做我的常规早晨简报。
```

**好的提示语：**
```
搜索有关 AI 智能体和开源大语言模型的最新新闻。
找到过去 24 小时内至少 5 篇最近的文章。
总结最重要的 3 条新闻，并以简洁的每日简报格式呈现。
每条新闻包括：清晰标题、两句话摘要和来源 URL。
使用友好、专业的语气。
使用表情符号项目符号格式化。
```

好的提示语具体说明了**要搜索的内容**、**文章数量**、**格式要求**和**语气风格**。这是一次性提供智能体所需的所有信息。

## 步骤 3：自定义简报

一旦基本的简报正常工作，您就可以发挥创意了。

### 多主题简报

在一个简报中涵盖多个领域：

```
/cron add "0 8 * * *" "创建一个涵盖三个主题的早晨简报。
对每个主题，搜索过去 24 小时内的最新新闻，并总结最重要的 2 条新闻及其链接。

主题：
1. AI 和机器学习 — 重点关注开源模型和智能体框架
2. 加密货币 — 重点关注比特币、以太坊和监管新闻
3. 太空探索 — 重点关注 SpaceX、NASA 和商业航天

使用清晰的章节标题和表情符号格式化。以今天的日期和励志语录结尾。"
```

### 使用委托进行并行研究

为了更快的简报，让 Hermes 将每个主题委托给子智能体：

```
/cron add "0 8 * * *" "通过委托研究创建早晨简报。
将三个并行任务委托给子智能体：

1. 委托：搜索过去 24 小时内最重要的 2 条 AI/ML 新闻及其链接
2. 委托：搜索过去 24 小时内最重要的 2 条加密货币新闻及其链接
3. 委托：搜索过去 24 小时内最重要的 2 条太空探索新闻及其链接

收集所有结果，将它们合并为一个整洁的简报，包含章节标题、表情符号格式化和来源链接。
添加今天的日期作为标题。"
```

每个子智能体独立并行搜索，然后主智能体将所有内容组合成一份精美的简报。
有关此工作原理的更多信息，请参见 [委托文档](/docs/user-guide/features/delegation)。

### 仅工作日计划

不需要周末简报？使用针对周一到周五的定时表达式：

```
/cron add "0 8 * * 1-5" "搜索最新的 AI 和科技新闻..."
```

### 每日两次简报

获取早晨概览和晚间回顾：

```
/cron add "0 8 * * *" "早晨简报：搜索过去 12 小时内的 AI 新闻..."
/cron add "0 18 * * *" "晚间回顾：搜索过去 12 小时内的 AI 新闻..."
```

### 添加个人上下文记忆

如果您启用了[记忆功能](/docs/user-guide/features/memory)，可以存储跨会话持久化的偏好设置。但请记住——定时任务在新会话中运行，没有对话记忆。要在提示语中添加个人上下文，请直接将其嵌入其中：

```
/cron add "0 8 * * *" "您正在为一位关心以下内容的资深 ML 工程师创建简报：
PyTorch 生态系统、Transformer 架构、开源权重模型和欧盟 AI 监管。
跳过产品发布或融资轮次新闻，除非涉及开源内容。

搜索这些主题的最新新闻。总结最重要的 3 条新闻及其链接。
保持简洁和技术性——这位读者不需要基础解释。"
```

:::tip 定制人物设定
包含简报*面向对象*的详细信息会显著提高相关性。告诉智能体您的角色、兴趣和要跳过的内容。
:::

## 步骤 4：管理您的任务

### 列出所有计划的任务

在聊天中：
```
/cron list
```

或在终端中：
```bash
hermes cron list
```

您将看到如下输出：

```
ID          | 名称              | 计划        | 下次运行时间         | 发送目标
------------|-------------------|-------------|--------------------|--------
a1b2c3d4    | 早晨简报          | 0 8 * * *   | 2026-03-09 08:00   | telegram
e5f6g7h8    | 晚间回顾          | 0 18 * * *  | 2026-03-08 18:00   | telegram
```

### 删除任务

在聊天中：
```
/cron remove a1b2c3d4
```

或对话式询问：
```
删除我的早晨简报定时任务。
```

Hermes 将使用 `cronjob(action="list")` 查找它，并使用 `cronjob(action="remove")` 删除它。

### 检查 Gateway 状态

确保调度程序确实在运行：

```bash
hermes cron status
```

如果 Gateway 没有运行，您的任务将无法执行。将其安装为背景服务以确保可靠性：

```bash
hermes gateway install
# 或在 Linux 服务器上
sudo hermes gateway install --system
```
