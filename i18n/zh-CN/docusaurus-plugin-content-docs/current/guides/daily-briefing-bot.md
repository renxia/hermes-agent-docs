# 教程：构建一个每日简报机器人

在本教程中，你将构建一个个人简报机器人，它每天早上唤醒，研究你关心的主题，总结发现，并将简洁的简报直接发送到你的 Telegram 或 Discord。

最后，你将拥有一个完全自动化的工作流，结合了**网页搜索**、**定时任务调度**、**任务委派**和**消息传递** —— 无需编写任何代码。

## 我们在构建什么

以下是流程：

1. **上午 8:00** — 定时任务调度器触发你的作业
2. **Hermes 启动**一个全新的智能体会话，加载你的提示词
3. **网页搜索**获取关于你主题的最新消息
4. **总结提炼**将其提炼成简洁的简报格式
5. **分发传递**将简报发送到你的 Telegram 或 Discord

整个过程全自动运行。你只需在晨间喝咖啡时阅读简报即可。

## 前提条件

在开始之前，请确保您已具备：

- **Hermes 智能体已安装** — 参见[安装指南](/getting-started/installation)
- **网关正在运行** — 网关守护进程负责处理定时任务执行：
  ```bash
  hermes gateway install   # 作为用户服务安装
  sudo hermes gateway install --system   # Linux服务器：系统引导服务
  # 或者
  hermes gateway           # 前台运行
  ```
- **Firecrawl API 密钥** — 在环境变量中设置 `FIRECRAWL_API_KEY` 用于网络搜索
- **消息功能已配置**（可选但推荐） — 已通过[Telegram](/user-guide/messaging/telegram)或Discord设置主频道

:::tip 没有消息功能？没问题
您仍然可以使用 `deliver: "local"` 来遵循本教程。简报将保存至 `~/.hermes/cron/output/` 目录，您可以随时阅读。
:::

## 步骤 1：手动测试工作流

在自动化之前，我们先确保简报功能正常工作。启动一个聊天会话：

```bash
hermes
```

然后输入以下提示：

```
搜索关于AI智能体和开源LLM的最新新闻。
总结前3条重要消息，以简洁的简报格式呈现并附上链接。
```

Hermes 将搜索网络、浏览结果，并生成类似这样的内容：

```
☀️ 您的AI简报 — 2026年3月8日

1. 通义千问3发布，拥有2350亿参数
   阿里巴巴最新的开源权重模型在多项基准测试中达到了GPT-4.5的水平，同时保持了完全开源。
   → https://qwenlm.github.io/blog/qwen3/

2. LangChain推出智能体协议标准
   一种新的智能体间通信开放标准在发布第一周就获得了15个主要框架的支持。
   → https://blog.langchain.dev/agent-protocol/

3. 欧盟《人工智能法案》开始针对通用模型执法
   首批合规截止日期生效，开源模型在1000万参数阈值下获得豁免。
   → https://artificialintelligenceact.eu/updates/

---
3条消息 • 搜索来源：8 • 由Hermes智能体生成
```

如果此功能正常，您就可以准备自动化了。

:::tip 优化格式
尝试不同的提示词，直到得到您满意的输出。添加诸如“使用表情符号标题”或“每条摘要不超过2句话”之类的指令。您最终确定的格式将用于定时任务。
:::

## 步骤 2：创建定时任务

现在我们来安排每天早上自动执行此任务。您可以通过两种方式实现。

在创建定时任务之前，请确保Hermes已全局配置默认模型和提供商。如果您希望特定任务使用不同的值，请在创建时设置显式的、针对每个任务的模型/提供商覆盖。

### 方式 A：自然语言（在聊天中）

直接告诉Hermes您想要什么：

```
每天早上8点，搜索网络查找关于AI智能体和开源LLM的最新新闻。总结前3条重要消息，以简洁的简报形式呈现并附上链接。使用友好、专业的语气。发送至Telegram。
```

Hermes将使用统一的 `cronjob` 工具为您创建定时任务。

### 方式 B：CLI 斜杠命令

使用 `/cron` 命令以获得更多控制：

```
/cron add "0 8 * * *" "搜索网络查找关于AI智能体和开源LLM的最新新闻。从过去24小时内找到至少5篇最近文章。将前3个最重要消息总结为简洁的每日简报格式。每条消息包括：一个清晰的标题、一个2句话的摘要和来源URL。使用友好、专业的语气。使用表情符号项目符号格式化，并以消息总数结尾。"
```

### 黄金法则：自包含的提示词

:::warning 关键概念
定时任务在**全新的会话**中运行——没有您之前对话的记忆，也没有关于您“之前设置”的上下文。您的提示词必须包含智能体完成工作所需的**所有信息**。
:::

**糟糕的提示词：**
```
执行我平时的早间简报。
```

**良好的提示词：**
```
搜索网络查找关于AI智能体和开源LLM的最新新闻。从过去24小时内找到至少5篇最近文章。将前3个最重要消息总结为简洁的每日简报格式。每条消息包括：一个清晰的标题、一个2句话的摘要和来源URL。使用友好、专业的语气。使用表情符号项目符号格式化。
```

良好的提示词在**搜索什么**、**文章数量**、**格式**和**语气**方面都是具体的。它一次性提供了智能体所需的所有信息。

## 步骤 3：自定义简报

基本简报正常工作后，您可以发挥创意。

### 多主题简报

在一份简报中涵盖多个领域：

```
/cron add "0 8 * * *" "创建一份涵盖三个主题的早间简报。对于每个主题，搜索过去24小时内的网络新闻，并总结前2条重要消息及链接。

主题：
1. 人工智能与机器学习 — 重点关注开源模型和智能体框架
2. 加密货币 — 重点关注比特币、以太坊和监管新闻
3. 太空探索 — 重点关注SpaceX、NASA和商业航天

格式为清晰的简报，带有章节标题和表情符号。以今天的日期和一句励志名言结尾。"
```

### 使用委托进行并行研究

为了更快生成简报，可以告诉Hermes将每个主题委托给子智能体：

```
/cron add "0 8 * * *" "通过委托研究给子智能体来创建早间简报。委托三个并行任务：

1. 委托：搜索过去24小时内关于AI/ML的前2条新闻消息及链接
2. 委托：搜索过去24小时内关于加密货币的前2条新闻消息及链接
3. 委托：搜索过去24小时内关于太空探索的前2条新闻消息及链接

收集所有结果，并将它们组合成一份简洁的简报，带有章节标题、表情符号格式和来源链接。添加今天的日期作为标题。"
```

每个子智能体独立并行搜索，然后主智能体将所有内容整合成一份精美的简报。有关此工作原理的更多信息，请参阅[委托文档](/user-guide/features/delegation)。

### 仅工作日安排

周末不需要简报？使用针对周一至周五的定时表达式：

```
/cron add "0 8 * * 1-5" "搜索关于AI和科技的最新新闻..."
```

### 每日两次简报

获取早间概览和晚间回顾：

```
/cron add "0 8 * * *" "早间简报：搜索过去12小时内的AI新闻..."
/cron add "0 18 * * *" "晚间回顾：搜索过去12小时内的AI新闻..."
```

### 使用记忆功能添加个人上下文

如果您启用了[记忆功能](/user-guide/features/memory)，您可以存储跨会话持久存在的偏好设置。但请记住——定时任务在没有对话记忆的新会话中运行。要添加个人上下文，请将其直接写入提示词：

```
/cron add "0 8 * * *" "您正在为一位高级机器学习工程师创建简报，他关心：PyTorch生态系统、transformer架构、开源权重模型以及欧盟的AI监管。除非涉及开源，否则跳过关于产品发布或融资的消息。

搜索关于这些主题的最新新闻。总结前3条消息及链接。简洁且技术性强——这位读者不需要基础解释。"
```

:::tip 量身定制角色
包含简报是*为谁*准备的详细信息，可以显著提高相关性。告诉智能体您的角色、兴趣以及需要跳过的内容。
:::

## 步骤 4：管理您的任务

### 列出所有计划任务

在聊天中：
```
/cron list
```

或在终端中：
```bash
hermes cron list
```

您将看到类似以下的输出：

```
ID          | 名称             | 计划        | 下次运行           | 投递方式
------------|------------------|-------------|--------------------|--------
a1b2c3d4    | 早间简报         | 0 8 * * *   | 2026-03-09 08:00   | telegram
e5f6g7h8    | 晚间回顾         | 0 18 * * *  | 2026-03-08 18:00   | telegram
```

### 移除任务

在聊天中：
```
/cron remove a1b2c3d4
```

或者通过对话方式询问：
```
移除我的早间简报定时任务。
```

Hermes将使用 `cronjob(action="list")` 查找它，并使用 `cronjob(action="remove")` 删除它。

### 检查网关状态

确保调度程序确实在运行：

```bash
hermes cron status
```

如果网关没有运行，您的任务将不会执行。将其作为后台服务安装以提高可靠性：

```bash
hermes gateway install
# 或在Linux服务器上
sudo hermes gateway install --system
```
