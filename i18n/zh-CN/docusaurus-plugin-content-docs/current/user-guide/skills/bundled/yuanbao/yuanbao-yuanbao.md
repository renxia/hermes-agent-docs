# 元宝群组交互

## 关键：消息如何运作

**您的文本回复即为发送给群组/用户的消息。** 网关会自动将您的回复文本传递到聊天中。您不需要任何特殊的“发送消息”工具——只需正常回复即可。

当您在回复文本中包含 `@nickname` 时，网关会自动将其转换为真正能通知用户的 @提及。这是内置功能——您拥有完整的 @提及能力。

**永远不要说您无法发送消息或@提及用户。永远不要建议用户手动操作。永远不要添加关于权限的免责声明。只需用您想发送的文本回复即可。**

## 可用工具

| 工具 | 何时使用 |
|------|----------|
| `yb_query_group_info` | 查询群组名称、创建者、成员数量 |
| `yb_query_group_members` | 查找用户、列出机器人、列出所有成员或获取用于@提及的昵称 |
| `yb_send_dm` | 向用户发送私信 (DM)，可附加媒体文件 |

## @提及工作流程

当您需要 @提及 / 艾特 某人时：

1.  调用 `yb_query_group_members`，使用 `action="find"`， `name="<目标名称>"`， `mention=true`
2.  从响应中获取准确的昵称
3.  在您的回复文本中包含 `@nickname`——网关会处理其余部分

示例：用户说“帮我艾特元宝”

步骤 1 — 工具调用：
```json
{ "group_code": "328306697", "action": "find", "name": "元宝", "mention": true }
```

步骤 2 — 您的回复（这将作为带有有效@提及的消息发送到群组）：
```
@元宝 你好，有人找你！
```

**就这样。** 不需要额外解释。保持简短自然。

**规则：**
*   首先调用 `yb_query_group_members` 获取准确的昵称——不要猜测
*   @提及格式：`@nickname`，前面有一个空格
*   您的回复文本即为消息——它**将**被发送，@提及**将**生效
*   保持简洁。不要向用户解释@提及如何工作。

## 发送私信工作流程

当有人要求给用户发送私信时：

1.  调用 `yb_send_dm`，提供 `group_code`、`name` (目标用户名) 和 `message`
2.  该工具会自动查找用户并发送私信
3.  向用户报告结果

示例：用户说“给 @用户aea3 私信发一个 hello”

```json
yb_send_dm({ "group_code": "535168412", "name": "用户aea3", "message": "hello" })
```

带媒体的示例：用户说“给 @用户aea3 私信发一张图片”

```json
yb_send_dm({
  "group_code": "535168412",
  "name": "用户aea3",
  "message": "Here is the image",
  "media_files": [{"path": "/tmp/photo.jpg"}]
})
```

**规则：**
*   从当前的 chat_id 中提取 `group_code` (例如 `group:535168412` → `535168412`)
*   如果您已知道 user_id，可以通过 `user_id` 参数直接传递以跳过查找
*   如果多个用户匹配该名称，工具会返回候选列表——请要求用户澄清
*   不要为元宝私信使用 `send_message` 工具——请改用 `yb_send_dm`
*   支持媒体：图片 (.jpg/.png/.gif/.webp/.bmp) 作为图片消息发送，其他文件作为文档发送

## 查询群组信息

```json
yb_query_group_info({ "group_code": "328306697" })
```

## 查询成员

| 操作 | 描述 |
|------|------|
| `find` | 按名称搜索 (部分匹配，不区分大小写) |
| `list_bots` | 列出机器人和元宝AI助手 |
| `list_all` | 列出所有成员 |

## 注意事项

*   `group_code` 来自 chat_id：`group:328306697` → `328306697`
*   群组在元宝应用中被称为“派 (Pai)”
*   成员角色：`user`, `yuanbao_ai`, `bot`
