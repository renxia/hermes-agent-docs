# 宝可梦玩家

使用 `pokemon-agent` 包通过无头模拟器游玩宝可梦游戏。

## 何时使用
- 用户说“玩宝可梦”、“开始宝可梦”、“宝可梦游戏”
- 用户询问关于宝可梦红、蓝、黄、火红等版本
- 用户希望观看 AI 玩宝可梦
- 用户提及 ROM 文件（.gb、.gbc、.gba）

## 启动流程

### 1. 首次设置（克隆、虚拟环境、安装）
该仓库位于 GitHub 上的 NousResearch/pokemon-agent。克隆它，然后设置一个 Python 3.10+ 虚拟环境。使用 uv（推荐，速度更快）创建虚拟环境并以可编辑模式安装带有 pyboy 额外依赖的包。如果 uv 不可用，则回退到 python3 -m venv + pip。

在此机器上，它已设置于 /home/teknium/pokemon-agent，并准备好了虚拟环境——只需 cd 到该目录并执行 source .venv/bin/activate。

你还需要一个 ROM 文件。向用户索取他们的 ROM。在此机器上，该目录内存在一个 roms/pokemon_red.gb。
切勿下载或提供 ROM 文件——始终向用户索取。

### 2. 启动游戏服务器
在激活虚拟环境的 pokemon-agent 目录内，运行 pokemon-agent serve，并使用 --rom 指向 ROM，--port 9876。使用 & 在后台运行。
要从保存的游戏恢复，添加 --load-state 及存档名称。
等待 4 秒启动，然后通过 GET /health 验证。

### 3. 为用户设置实时仪表板以供观看
使用通过 localhost.run 的 SSH 反向隧道，以便用户可以在浏览器中查看仪表板。使用 ssh 连接，将本地端口 9876 转发到 nokey@localhost.run 的远程端口 80。将输出重定向到日志文件，等待 10 秒，然后从日志中 grep 出 .lhr.life URL。将带有 /dashboard/ 后缀的 URL 提供给用户。
隧道 URL 每次都会变化——如果重启，请向用户提供新的 URL。

## 保存与加载

### 何时保存
- 每 15-20 回合游戏后
- 在道馆战、劲敌遭遇或高风险战斗前**务必**保存
- 进入新城镇或地牢前
- 在任何你不确定的行动前

### 如何保存
POST /save 并附带描述性名称。良好示例：
before_brock、route1_start、mt_moon_entrance、got_cut

### 如何加载
POST /load 并附带存档名称。

### 列出可用存档
GET /saves 返回所有已保存状态。

### 服务器启动时加载
启动服务器时使用 --load-state 标志以自动加载存档。
这比启动后通过 API 加载更快。

### 步骤 1：观察 — 检查状态并截图
GET /state 获取位置、HP、战斗、对话框信息。
GET /screenshot 并保存到 /tmp/pokemon.png，然后使用 vision_analyze。
始终执行**两者**——内存状态提供数值，视觉提供空间感知。

## 动作参考
- press_a — 确认、交谈、选择
- press_b — 取消、关闭菜单
- press_start — 打开游戏菜单
- walk_up/down/left/right — 移动一格
- hold_b_N — 按住 B 键 N 帧（用于加速跳过文本）
- wait_60 — 等待约 1 秒（60 帧）
- a_until_dialog_end — 重复按 A 直到对话框清除

## 来自经验的关键提示

### 持续使用视觉
- 每 2-4 步移动后截图
- 内存状态告诉你位置和 HP，但**不**告诉你周围有什么
- 悬崖、围栏、标志、建筑物门、NPC — 只能通过截图看到
- 向视觉模型提出具体问题：“我北边一格是什么？”
- 当卡住时，在尝试随机方向前务必截图

### 传送过渡需要额外等待时间
当穿过门或楼梯时，屏幕会在地图过渡期间淡出为黑色。你**必须**等待其完成。在任何门/楼梯传送后添加 2-3 个 wait_60 动作。不等待会导致位置读取为过时，你会认为你仍在旧地图中。

### 建筑物出口陷阱
当你离开建筑物时，你会直接出现在门**前方**。如果你向北走，你会直接回到里面。**务必**先侧移，向左或向右走 2 格，然后按你的预期方向前进。

### 对话框处理
第一世代文本逐字母缓慢滚动。要加速跳过对话框，按住 B 键 120 帧，然后按 A。根据需要重复。按住 B 会使文本以最大速度显示。然后按 A 前进到下一行。
a_until_dialog_end 动作检查内存中的对话框标志，但此标志**无法**捕获所有文本状态。如果对话框似乎卡住，请改用手动 hold_b + press_a 模式，并通过截图验证。

### 悬崖是单向的
悬崖（小悬崖边缘）只能向下跳（南），**不能**向上爬（北）。如果被向北的悬崖阻挡，你必须向左或向右走以找到绕过它的缺口。使用视觉识别缺口在哪个方向。明确询问视觉模型。

### 导航策略
- 一次移动 2-4 步，然后截图检查位置
- 进入新区域时，立即截图以定向
- 询问视觉模型“到 [目的地] 的方向是？”
- 如果卡住 3 次以上，截图并完全重新评估
- 不要连续发送 10-15 次移动 — 你会 overshoot 或卡住

### 从野生战斗中逃跑
在战斗菜单中，RUN（逃跑）在右下角。要从默认光标位置（FIGHT，左上角）到达它：按下然后右移光标到 RUN，然后按 A。用 hold_b 包裹以加速跳过文本/动画。

### 战斗（FIGHT）
在战斗菜单中，FIGHT（战斗）在左上角（默认光标位置）。
按 A 进入招式选择，再次按 A 使用第一个招式。
然后按住 B 以加速跳过攻击动画和文本。

## 战斗策略

### 决策树
1. 想捕捉？→ 削弱然后投掷精灵球
2. 不需要的野生宝可梦？→ 逃跑
3. 属性优势？→ 使用效果绝佳的招式
4. 无优势？→ 使用最强 STAB 招式
5. HP 低？→ 切换或使用药水

### 第一世代属性表（关键 matchup）
- 水克火、地面、岩石
- 火克草、虫、冰
- 草克水、地面、岩石
- 电克水、飞行
- 地面克火、电、岩石、毒
- 超能克格斗、毒（在第一世代中 dominant！）

### 第一世代特性
- 特殊属性 = 特殊招式的攻击**和**防御
- 超能属性过强（幽灵招式有 bug）
- 暴击基于速度属性
- 绑紧/束缚阻止对手行动
- 集中精神 bug：**降低**暴击率而非提高

## 停止游戏
1. 通过 POST /save 使用描述性名称保存游戏
2. 使用 PKM:PROGRESS 更新内存
3. 告诉用户：“游戏已保存为 [名称]！说‘玩宝可梦’以恢复。”
4. 杀死服务器和隧道后台进程
