# 创造性构思

通过创造性约束生成项目创意。约束 + 方向 = 创造力。

## 工作原理

1. **从下方库中选择一个约束** — 随机选择，或匹配用户的领域/心情
2. **广义解释它** — 一个编码提示可以变成一个硬件项目，一个艺术提示可以变成一个 CLI 工具
3. **生成 3 个满足该约束的具体项目创意**
4. **如果他们选了一个，就构建它** — 创建项目，编写代码，发布它

## 规则

每个提示都应尽可能广义地解释。“这包括 X 吗？” → 是的。提示提供方向和轻微约束。缺少其中任何一个，就没有创造力。

## 约束库

### 面向开发者

**解决你自己的痛点：**
构建你本周希望存在的工具。少于 50 行代码。今天就发布它。

**自动化烦人的事情：**
你工作流程中最繁琐的部分是什么？用脚本来解决它。用两小时修复一个每天浪费你五分钟的问题。

**应该存在的 CLI 工具：**
想想你曾希望你能输入的命令。`git undo-that-thing-i-just-did`。`docker why-is-this-broken`。`npm explain-yourself`。现在构建它。

**除了胶水外没有新东西：**
完全用现有的 API、库和数据集制作一些东西。唯一原创的贡献是你如何连接它们。

**弗兰肯斯坦周：**
取一个做 X 的东西，让它做 Y。一个播放音乐的 git 仓库。一个生成诗歌的 Dockerfile。一个发送赞美的 cron 任务。

**减法：**
在代码库崩溃前，你能移除多少？将一个工具剥离到其最小可行功能。删除直到只剩下本质。

**高概念，低投入：**
一个深刻的想法，懒散地执行。概念应该是 brilliant 的。实现应该只需一个下午。如果需要更长时间，说明你想得太复杂了。

### 面向创客与艺术家

**公然复制某物：**
选择你钦佩的东西 — 一个工具、一件艺术品、一个界面。从头开始重新创建它。学习在于你的版本与他们之间的差距。

**一百万个某物：**
一百万既是很多，也没那么多。一百万像素是一张 1MB 的照片。一百万次 API 调用是周二的事。一百万个任何东西在规模上都会变得有趣。

**制作一个会“死”的东西：**
一个每天失去一个功能的网站。一个会遗忘的聊天机器人。一个倒数到虚无的倒计时。一个关于腐烂、杀戮或放手的练习。

**做很多数学：**
生成几何、着色器高尔夫、数学艺术、计算折纸。是时候重新学习 arcsin 是什么了。

### 面向任何人

**文本是通用接口：**
构建一个文本是唯一接口的东西。没有按钮，没有图形，只有输入和输出的文字。文本几乎可以进出任何东西。

**从笑点开始：**
想一个有趣的句子。倒推使其成为现实。“我教我的恒温器对我进行煤气灯操纵” → 现在构建它。

**敌对 UI：**
制作一个故意难以使用的东西。一个需要 47 个条件的密码字段。一个每个标签都在撒谎的表单。一个评判你命令的 CLI。

**再做一次：**
记住一个旧项目。从头开始再做一次。不要看原版。看看你的思维方式有什么变化。

参见 `references/full-prompt-library.md` 以获取 30 多个额外的约束，涵盖沟通、规模、哲学、转换等方面。

## 将约束匹配给用户

| 用户说 | 从中选择 |
|-----------|-----------|
| “我想做点东西”（无方向） | 随机 — 任何约束 |
| “我在学习 [语言]” | 公然复制某物，自动化烦人的事情 |
| “我想要奇怪的东西” | 敌对 UI，弗兰肯斯坦周，从笑点开始 |
| “我想要有用的东西” | 解决你自己的痛点，应该存在的 CLI 工具，自动化烦人的事情 |
| “我想要美丽的东西” | 做很多数学，一百万个某物 |
| “我精疲力尽了” | 高概念低投入，制作一个会“死”的东西 |
| “周末项目” | 除了胶水外没有新东西，从笑点开始 |
| “我想要挑战” | 一百万个某物，减法，再做一次 |

## 输出格式

```
## 约束：[名称]
> [约束，一句话]

### 创意

1. **[一句话简介]**
   [2-3 句话：你会构建什么以及为什么它有趣]
   ⏱ [周末 / 周 / 月] • 🔧 [技术栈]

2. **[一句话简介]**
   [2-3 句话]
   ⏱ ... • 🔧 ...

3. **[一句话简介]**
   [2-3 句话]
   ⏱ ... • 🔧 ...
```

## 示例

```
## 约束：应该存在的 CLI 工具
> 想想你曾希望你能输入的命令。现在构建它。

### 创意

1. **`git whatsup` — 显示你离开时发生了什么**
   将你的最后一次活跃提交与 HEAD 进行比较，并总结发生了什么变化、
   谁提交了、以及哪些 PR 合并了。就像来自你仓库的晨会站会。
   ⏱ 周末 • 🔧 Python, GitPython, click

2. **`explain 503` — 面向人类的 HTTP 状态码**
   管道传输任何状态码或错误消息，并获得通俗易懂的解释，
   包括常见原因和修复方法。从 curated 数据库中提取，而非 LLM。
   ⏱ 周末 • 🔧 Rust 或 Go, 静态数据集

3. **`deps why <package>` — 为什么这个包在我的依赖树中**
   追溯一个传递依赖，找到引入它的直接依赖。
   用一个命令回答“为什么我有 47 个 lodash 副本”。
   ⏱ 周末 • 🔧 Node.js, npm/yarn lockfile 解析
```

用户选择一个后，开始构建 — 创建项目，编写代码，迭代。
