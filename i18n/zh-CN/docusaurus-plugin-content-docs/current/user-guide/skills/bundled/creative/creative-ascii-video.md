# ASCII 视频制作流程

## 创意标准

这是视觉艺术。ASCII 字符是媒介；电影是标准。

**在编写一行代码之前**，请先阐明创意概念。情绪是什么？这讲述了什么视觉故事？**这个项目**与其他所有 ASCII 视频有何不同？用户的提示是一个起点——要用创意雄心来诠释它，而不是字面转述。

**首次渲染的卓越性是不可协商的。** 输出必须具有视觉冲击力，无需修改。如果看起来普通、平淡或像“AI 生成的 ASCII 艺术”，那就是错误的——在发布之前请重新思考创意概念。

**超越参考词汇。** 参考中的效果目录、着色器预设和调色板库是起始词汇。对于每个项目，请组合、修改并发明新的图案。目录是颜料盘——你创作的是绘画。

**主动创意。** 当项目需要时，请扩展技能的词汇。如果参考中没有满足愿景的内容，请自行构建。至少包含一个用户未要求但会欣赏的视觉时刻——一个转场、一个效果、一个提升整体作品的色彩选择。

**整体美学优于技术正确性。** 视频中的所有场景必须通过统一的视觉语言（共享的色温、相关的字符调色板、一致的运动词汇）感觉相互关联。每个场景都使用随机不同效果的技术上正确的视频是美学上的失败。

**密集、分层、深思熟虑。** 每一帧都应值得观看。永远不要使用纯黑背景。始终使用多网格构图。始终进行逐场景变化。始终有意图的色彩。

## 技术栈

每个项目一个独立的 Python 脚本。无需 GPU。

| 层级 | 工具 | 用途 |

| 核心 | Python 3.10+、NumPy | 数学、数组操作、向量化效果 |
| 信号 | SciPy | FFT、峰值检测（音频模式） |
| 图像处理 | Pillow (PIL) | 字体光栅化、帧解码、图像 I/O |
| 视频 I/O | ffmpeg (CLI) | 解码输入、编码输出、混合音频 |
| 并行 | concurrent.futures | 用于批量/片段渲染的 N 个工作者 |
| TTS | ElevenLabs API（可选） | 生成旁白片段 |
| 可选 | OpenCV | 视频帧采样、边缘检测 |

## 流程架构

每种模式都遵循相同的 6 阶段流程：

```
输入 → 分析 → 场景函数 → 色调映射 → 着色 → 编码
```

1. **输入** — 加载/解码源素材（视频帧、音频样本、图像或无）
2. **分析** — 提取每帧特征（音频频段、视频亮度/边缘、运动向量）
3. **场景函数** — 场景函数渲染到像素画布（`uint8 H,W,3`）。通过 `_render_vf()` + 像素混合模式组合多个字符网格。参见 `references/composition.md`
4. **色调映射** — 基于百分位的自适应亮度归一化。参见 `references/composition.md` § 自适应色调映射
5. **着色** — 通过 `ShaderChain` + `FeedbackBuffer` 进行后处理。参见 `references/shaders.md`
6. **编码** — 将原始 RGB 帧通过管道传输到 ffmpeg 进行 H.264/GIF 编码

## 关键实现注意事项

### 亮度 — 使用 `tonemap()`，而非线性乘法

这是头号视觉问题。黑色背景上的 ASCII 字符天生较暗。**切勿使用 `canvas * N` 乘法** — 这会导致高光被截断。请使用自适应色调映射：

```python
def tonemap(canvas, gamma=0.75):
    f = canvas.astype(np.float32)
    lo, hi = np.percentile(f[::4, ::4], [1, 99.5])
    if hi - lo < 10: hi = lo + 10
    f = np.clip((f - lo) / (hi - lo), 0, 1) ** gamma
    return (f * 255).astype(np.uint8)
```

流水线：`scene_fn() → tonemap() → FeedbackBuffer → ShaderChain → ffmpeg`

每个场景的伽马值：默认 0.75，日晒效果 0.55，海报化 0.50，明亮场景 0.85。对于暗层，使用 `screen` 混合模式（而非 `overlay`）。

### 字体单元格高度

macOS Pillow：`textbbox()` 返回的高度不正确。请使用 `font.getmetrics()`：`cell_height = ascent + descent`。参见 `references/troubleshooting.md`。

### ffmpeg 管道死锁

切勿对长时间运行的 ffmpeg 使用 `stderr=subprocess.PIPE` — 缓冲区达到 64KB 后会死锁。请重定向到文件。参见 `references/troubleshooting.md`。

### 字体兼容性

并非所有 Unicode 字符都能在所有字体中渲染。请在初始化时验证调色板 — 渲染每个字符，检查是否输出空白。参见 `references/troubleshooting.md`。

### 按片段架构

对于分段视频（引文、场景、章节），将每个片段渲染为单独的文件，以实现并行渲染和选择性重新渲染。参见 `references/scenes.md`。
