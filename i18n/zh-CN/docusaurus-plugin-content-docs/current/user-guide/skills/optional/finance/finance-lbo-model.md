## 关键指令 — 请先阅读

使用 Python/openpyxl。编写公式字符串（`ws["D20"] = "=B5*B6"`），然后在交付前运行 `excel-author` 技能的 `recalc.py` 辅助程序。

### 核心原则
* **每个计算必须是 Excel 公式** - 绝不在 Python 中计算值并将结果硬编码到单元格中。使用 openpyxl 时，请编写 `cell.value = "=B5*B6"`（公式字符串），而不是 `cell.value = 1250`（计算结果）。模型必须是动态的，并在输入更改时更新。
* **使用模板结构** - 遵循 `examples/LBO_Model.xlsx` 或用户提供的模板中的组织结构。不要自行发明布局。
* **使用正确的单元格引用** - 所有公式都应引用相应的单元格。绝不键入应来自其他单元格的数字。
* **保持符号约定一致性** - 遵循模板使用的任何符号约定（一些使用负数表示现金流出，一些使用正数）。在整个过程中保持一致。
* **逐节工作，每一步都与用户确认** - 完整地完成一个部分，向用户展示已构建的内容，运行该部分的验证检查，并在**移动到下一个部分之前**获得确认。不要构建整个模型后一次性呈现——后面的部分依赖于前面的部分，因此在构建回报后才发现资金来源与用途中的错误意味着所有地方都需要返工。

### 公式颜色约定
* **蓝色 (0000FF)**：硬编码的输入值 - 不引用其他单元格的输入数字
* **黑色 (000000)**：包含计算的公式 - 使用运算符或函数的任何公式（`=B4*B5`，`=SUM()`，`=-MAX(0,B4)`）
* **紫色 (800080)**：链接到**同一标签页**上的单元格 - 没有计算的直接引用（`=B9`，`=B45`）
* **绿色 (008000)**：链接到**不同标签页**上的单元格 - 跨表引用（`=Assumptions!B5`，`='Operating Model'!C10`）

### 填充色板 — 专业蓝与灰（除非用户/模板另行指定，否则为默认）
* **保持简洁** — 仅使用蓝色和灰色进行单元格填充。**不要**引入绿色、黄色、红色或多种强调色。专业的 LBO 模型讲究克制。
* **默认填充色板：**
  * **章节标题**（资金来源与用途、运营模型等）：深蓝色 `#1F4E79`，白色粗体文本
  * **列标题**（第 1 年、第 2 年等）：浅蓝色 `#D9E1F2`，黑色粗体文本
  * **输入单元格**：浅灰色 `#F2F2F2`（或白色）——蓝色*字体*才是信号，填充色是次要的
  * **公式/计算单元格**：白色，无填充
  * **关键输出**（IRR、MOIC、退出权益）：中蓝色 `#BDD7EE`，黑色粗体文本
* **这就是整个色板。** 3 种蓝色 + 1 种灰色 + 白色。如果模板使用自己的颜色，请改为遵循模板。
* 注意：上面的蓝/黑/紫/绿**字体**颜色是为了区分输入、公式和链接。它们与这里的**填充**色板是分开的，两者共同工作。

### 数字格式标准
* **货币**：`$#,##0;($#,##0);"-"` 或 `$#,##0.0`，取决于模板
* **百分比**：`0.0%`（一位小数）
* **倍数**：`0.0"x"`（一位小数）
* **MOIC/详细比率**：`0.00"x"`（两位小数以提高精度）
* **所有数字单元格**：右对齐

## 模板分析阶段 — 请先执行此步骤

在填充任何公式之前，请彻底检查模板：

1. **映射结构** - 确定每个部分所在的位置以及它们如何相互关联。注意哪些部分为其他部分提供数据。

2. **理解时间线** - 哪些列代表哪些期间？是否有“收盘”或“备考”列？预测期从何处开始？

6. **注意模板特定的约定** - 符号约定、小计结构、部分如何组织、不同组件是否有单独的标签页等。

## 常见问题领域

以下计算模式在 LBO 模型中经常引起问题。当您遇到这些模式时，请特别注意：

### 平衡部分
* 当两个部分必须相等时（例如，资金来源 = 资金用途），通常有一个项目是“平衡项”（平衡数字）
* 确定哪个项目是平衡项，并将其计算为差额

### 税务计算
* 税务公式应仅引用相关的收入行和税率
* 不应引用不相关的部分（例如，债务计划）
* 考虑亏损是产生税盾还是简单地被忽略

### 利息和循环引用
* 如果利息计算引用了受现金流影响的余额，则可能导致循环
* 使用**期初余额**（而非平均或期末）来打破循环引用
* 模式：利息 → 现金流 → 偿还 → 期末余额（如果利息使用期末余额，则会循环回来）

### 债务偿还 / 现金清扫
* 当存在多个债务批次时，通常有优先顺序
* 现金清扫应遵守优先顺序
* 余额不能为负数 - 适当地使用 MAX 或 MIN 函数

### 回报计算 (IRR/MOIC)
* 现金流必须具有正确的符号：投资 = 负数，收益 = 正数
* 如果使用 XIRR，需要相应的日期
* 如果使用 IRR，现金流应在连续期间内
* MOIC = 总收益 / 总投资

### 敏感度表
* **使用奇数维度**（5×5 或 7×7）——绝不使用 4×4 或 6×6。奇数维度保证有真正的中心单元格。
* **中心单元格 = 基准情景。** 围绕模型的实际假设对称地构建行和列轴值（例如，如果基准进入倍数 = 10.0x，则轴 = `[8.0x, 9.0x, 10.0x, 11.0x, 12.0x]`）。然后，中心单元格的 IRR/MOIC **必须**等于模型实际输出的 IRR/MOIC——这证明表格连接正确。
* **突出显示中心单元格** — 中蓝色填充（`#BDD7EE`）+ 粗体字体，以便基准情景在视觉上被锚定。
* Excel 的 DATA TABLE 函数可能不适用于 openpyxl——请改为编写明确引用行/列标题的公式。
* 每个单元格应显示**不同的值**——如果全部相同，说明公式没有正确变化。
* 使用混合引用（例如，`$A5` 用于行输入，`B$4` 用于列输入）

## 应避免的常见错误

| 错误类型 | 可能出现的问题 | 修复方法 |
|----------|-----------------|----------|
| 硬编码计算值 | 输入更改时模型不更新 | 始终使用引用源单元格的公式 |
| 复制后单元格引用错误 | 公式指向错误的单元格 | 验证所有链接，适当使用 $ 锚定 |
| 循环引用错误 | 模型无法计算 | 利息类计算使用期初余额，打破循环 |
| 科目不平衡 | 应匹配的总计不匹配 | 确保一个项目是插入项（计算为差值） |
| 不可能出现的负余额 | 支付/使用超过可用金额 | 适当使用 MAX(0, ...) 或 MIN 函数 |
| IRR/回报错误 | 符号错误或范围不完整 | 检查现金流符号并确保公式覆盖所有期间 |
| 敏感性表格显示相同值 | 公式未随输入变化 | 检查单元格引用——需要混合引用 ($A5, B$4) |
| 滚动结转不一致 | 期初 ≠ 上期期末 | 验证期间之间的链接 |
| 符号惯例不一致 | 加法变成减法或反之 | 在整个模板中始终遵循其惯例 |
