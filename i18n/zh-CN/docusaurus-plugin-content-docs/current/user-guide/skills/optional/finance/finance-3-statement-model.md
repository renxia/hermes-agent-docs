# 三表财务模型模板填写

填写并生成整合的财务模型模板，确保利润表、资产负债表和现金流量表之间的正确链接。

## ⚠️ 关键原则 — 填写任何模板前必读

**公式优于硬编码（不可妥协）：**
- 每个预测单元格、滚动结转、链接和小计 **必须** 是Excel公式 —— 绝不能是预先计算的值
- 使用 Python/openpyxl 时：写入公式字符串 (`ws["D15"] = "=D14*(1+Assumptions!$B$5)"`)，而不是计算结果 (`ws["D15"] = 12500`)
- **唯一** 应包含硬编码数字的单元格是：(1) 历史实际数据，(2) 假设表中的驱动假设
- 如果您发现自己正在 Python 中计算某个值并将其写入单元格 —— **停止**。请改为写入公式。
- 原因：模型必须能够在情景切换或假设改变时灵活调整。硬编码会默默地破坏所有下游的完整性检查。

**与用户逐步验证：**
1. **模板映射后** → 向用户展示您识别出的标签页/部分，并在修改任何单元格前确认
2. **填写历史数据后** → 向用户展示历史数据块，并确认数值/期间与源数据匹配
3. **构建利润表预测后** → 运行小计检查，向用户展示预测利润表，确认后再进行资产负债表
4. **构建资产负债表后** → 向用户展示每个期间的平衡检查（资产 = 负债 + 权益），确认后再进行现金流量表
5. **构建现金流量表后** → 向用户展示现金勾稽（现金流期末现金 = 资产负债表现金），确认后再最终确定
6. **不要一次性填写整个模型然后呈现完整结果** — 在每张表处中断，展示工作成果，尽早发现错误

## 格式设置 — 专业蓝/灰色调色板（除非模板/用户另有指定，否则为默认）

**颜色保持简洁。** 单元格填充仅使用蓝色和灰色。**不要** 引入绿色、黄色、橙色或多种强调色 —— 一个整洁的模型注重克制。

| 元素 | 填充色 | 字体颜色 |
|---|---|---|
| 区域标题（IS / BS / CF 标题） | 深蓝 `#1F4E79` | 白色加粗 |
| 列标题（FY2024A, FY2025E 等） | 浅蓝 `#D9E1F2` | 黑色加粗 |
| 输入单元格（历史数据、假设驱动） | 浅灰 `#F2F2F2` 或白色 | 蓝色 `#0000FF` |
| 公式单元格 | 白色 | 黑色 |
| 跨表链接 | 白色 | 绿色 `#008000` |
| 检查行 / 关键合计 | 中蓝 `#BDD7EE` | 黑色加粗 |

**这就是 3 种蓝色 + 1 种灰色 + 白色。** 如果模板有自己的配色方案，请遵循模板。

字体颜色标识单元格 *性质*（输入/公式/链接）。填充颜色标识您 *所在位置*（标题/数据/检查）。

## 模型结构

### 识别模板标签页组织

模板的标签页命名惯例和组织方式各异。在填写之前，请检查所有标签页以了解模板结构。以下是常见的标签页名称及其典型内容：

| 常见标签页名称 | 应查找的内容 |
|------------------|----------------------|
| IS, P&L, Income Statement | 利润表 |
| BS, Balance Sheet | 资产负债表 |
| CF, CFS, Cash Flow | 现金流量表 |
| WC, Working Capital | 营运资本表 |
| DA, D&A, Depreciation, PP&E | 折旧与摊销表 |
| Debt, Debt Schedule | 债务表 |
| NOL, Tax, DTA | 净经营亏损表 |
| Assumptions, Inputs, Drivers | 驱动假设和输入 |
| Checks, Audit, Validation | 错误检查仪表板 |

**模板审查清单**
- 识别模板中存在的标签页（并非所有模板都包含所有附表）
- 注意任何未在上面列出的模板特定标签页
- 了解标签页之间的依赖关系（例如，哪些附表为主表提供数据）
- 在每个标签页上定位输入单元格与公式单元格

### 理解模板结构

在填写模板之前，熟悉其现有布局，以确保数据输入到正确位置且公式保持完整。

**识别行结构**
- 定位每个标签页顶部的模型标题
- 识别区域标题及其视觉分隔
- 找到单位行，指示 $ 百万、%、x 等
- 注意区分实际数据与预测数据期间的列标题
- 确认期间标签（例如，FY2024A，FY2025E）
- 识别输入单元格与公式单元格（通常通过字体颜色区分）

**识别列结构**
- 确认最左侧的行项目标签
- 验证历史年份在预测年份之前
- 注意区分历史期间与预测期间的视觉边框
- 检查所有标签页中列顺序的一致性

**使用命名范围**
模板通常使用命名范围来表示关键输入和输出。输入数据前：
- 审查模板中现有的命名范围（在 Excel 中：公式 → 名称管理器）
- 常见的命名范围包括：收入增长率、成本百分比、关键输出（净利润、EBITDA、总债务、现金）、情景选择器单元格
- 确保数据输入到这些命名范围所引用的单元格中

### 预测期间
- 模板通常从最后一个历史年份向前预测 5 年
- 验证历史（A）列与预测（E）列是否清晰分隔
- 确认列使用财年标注（例如，FY2024A，FY2025E）

## 利润率分析

**注意：以下利润率分析仅在用户提示或模板明确要求时执行。若无提示，请跳过此部分。**

计算并在利润表（IS）标签页上显示盈利能力利润率，以追踪运营效率并实现同行比较。

### 应包含的核心利润率

| 利润率 | 公式 | 衡量内容 |
|--------|---------|------------------|
| 毛利率 | 毛利润 / 收入 | 定价能力、生产效率 |
| EBITDA利润率 | EBITDA / 收入 | 核心经营盈利能力 |
| EBIT利润率 | EBIT / 收入 | 折旧摊销后的经营盈利能力 |
| 净利润率 | 净利润 / 收入 | 底线盈利能力 |

### 带利润率的利润表布局

在每个利润行项目下方直接显示利润率百分比：
- 毛利率 % 显示在毛利润下方
- EBIT 利润率 % 显示在 EBIT 下方
- EBITDA 利润率 % 显示在 EBITDA 下方
- 净利润率 % 显示在净利润下方

## 信用指标

**注意：以下信用分析仅在用户提示或模板明确要求时执行。若无提示，请跳过此部分。**

计算并在资产负债表（BS）标签页上显示信用/杠杆指标，以评估财务状况、债务容量和契约合规性。

### 应包含的核心信用指标

| 指标 | 公式 | 衡量内容 |
|--------|---------|------------------|
| 总债务 / EBITDA | 总债务 / 近12个月 EBITDA | 杠杆倍数 |
| 净债务 / EBITDA | （总债务 - 现金）/ 近12个月 EBITDA | 扣除现金后的杠杆 |
| 利息保障倍数 | EBITDA / 利息费用 | 偿债能力 |
| 债务 / 总资本 | 总债务 / （总债务 + 权益） | 资本结构 |
| 债务 / 权益 | 总债务 / 总权益 | 财务杠杆 |
| 流动比率 | 流动资产 / 流动负债 | 短期流动性 |
| 速动比率 | （流动资产 - 存货）/ 流动负债 | 即时流动性 |

### 信用指标层级检查

验证乐观情景显示出最强的信用状况：
- 杠杆率：乐观情景 &lt; 基准情景 &lt; 悲观情景（越低越好）
- 保障倍数：乐观情景 > 基准情景 > 悲观情景（越高越好）
- 流动性：乐观情景 > 基准情景 > 悲观情景（越高越好）

### 契约合规性跟踪

如果已知债务契约，请添加明确的合规性检查，将实际指标与契约阈值进行比较。

## 情景分析（基准 / 乐观 / 悲观）

在假设表中使用带有 CHOOSE 或 INDEX/MATCH 公式的情景切换（下拉菜单）。

| 情景 | 描述 |
|----------|-------------|
| 基准情景 | 管理层指引或一致预期 |
| 乐观情景 | 高于指引的增长、利润率扩张 |
| 悲观情景 | 低于趋势的增长、利润率压缩 |

**需敏感度分析的关键驱动因素**：收入增长率、毛利率、销售管理费用率（SG&A %）、应收账款周转天数（DSO）/存货周转天数（DIO）/应付账款周转天数（DPO）、资本支出占比（CapEx %）、利率、税率。

**情景审计检查**：切换情景影响所有报表，所有情景下资产负债表平衡，现金勾稽一致，层级关系成立（净利润、EBITDA、自由现金流、利润率方面：乐观情景 > 基准情景 > 悲观情景）。

## 完成模型模板

本部分提供完成任何三报表财务模型模板的通用指导，同时保留现有公式并确保数据完整性。

### 步骤1：分析模板结构

在输入任何数据之前，请彻底审查模板以理解其架构：

**识别输入单元格与公式单元格**
- 寻找视觉线索（字体颜色、单元格底色）来区分输入单元格和公式单元格
- 常见惯例：蓝色字体 = 输入，黑色字体 = 公式，绿色字体 = 链接到其他工作表
- 使用Excel的追踪引用/从属单元格（公式 → 追踪引用单元格）来理解单元格关系
- 检查可能控制关键输入的命名范围（公式 → 名称管理器）

**映射模板流程**
- 识别哪些工作表会输入到其他工作表（例如，假设 → 损益表 → 资产负债表 → 现金流量表）
- 注意任何支持性附表及其与主报表的链接
- 在填充数据之前，记录模板的具体行项目和结构

### 步骤2：不破坏公式地填充数据

**数据输入黄金法则**

| 规则 | 描述 |
|------|------|
| 只编辑输入单元格 | 除非有意替换公式，否则切勿覆盖包含公式的单元格 |
| 保留单元格引用 | 复制数据时，使用选择性粘贴（Ctrl+Shift+V）以避免用源格式覆盖公式 |
| 匹配模板的单位 | 在输入数据前，确认模板使用的是千元、百万元还是实际值 |
| 遵循符号惯例 | 遵循模板现有的符号惯例（例如，费用是正数还是负数） |
| 检查循环引用 | 如果模板使用迭代计算，请确保已启用“迭代计算”选项 |

**安全数据输入流程**
1.  识别指定的精确输入单元格（通常有高亮显示或标签）
2.  先输入历史数据，然后验证公式在这些期间计算正确
3.  输入用于预测计算的假设驱动因素
4.  审查计算结果，确认公式按预期工作
5.  如果必须修改公式单元格，请在更改前记录原始公式

**处理预建公式**
- 如果公式引用了您尚未填充的单元格，在完成所有输入之前，请预期会出现临时错误（#REF!、#DIV/0!）
- 当公式产生意外结果时，请追踪引用以识别缺失或不正确的输入
- 永远不要在未检查所有工作表中的公式依赖关系的情况下删除行/列

### 步骤3：验证公式

**公式完整性检查**

在依赖模板输出之前，请验证公式是否正确运行：

| 检查类型 | 方法 |
|----------|------|
| 追踪引用单元格 | 选择一个公式单元格 → 公式 → 追踪引用单元格，以验证它引用了正确的输入 |
| 追踪从属单元格 | 验证关键输入是否流向了预期的输出单元格 |
| 公式求值 | 使用 公式 → 公式求值 来逐步执行复杂计算 |
| 检查硬编码值 | 预测公式应引用假设，而不是包含硬编码值 |
| 用已知值测试 | 输入简单的测试值以验证公式产生预期结果 |
| 跨表一致性 | 确保相同的公式逻辑适用于所有预测期 |

**需要注意的常见公式问题**
- 混合绝对/相对引用，导致跨期间复制时结果不正确
- 指向外部文件或已删除范围的断开链接（#REF! 错误）
- 收入增长前的早期期间出现除以零错误（#DIV/0! 错误）
- 循环引用警告（可能对利息计算有意为之）
- 预测列之间的公式不一致（使用 Ctrl+\ 查找差异）

**验证跨表链接**
- 确认出现在多个工作表上的值是链接的（而不是重复的）
- 验证附表总计与主报表上的相应行项目一致
- 检查期间标签是否在所有工作表上对齐

### 步骤4：按工作表进行质量检查

填充模板后，对每个工作表执行以下验证检查：

**损益表（IS）质量检查**
- 历史期间的收入数据与源数据匹配
- 所有费用行项目总和等于报告总额
- 小计（毛利、EBIT、EBT、净利润）计算正确
- 税项计算逻辑合理（能正确处理亏损）
- 预测驱动因素引用了假设工作表（无硬编码）
- 期间间变化在方向上是合理的

**资产负债表（BS）质量检查**
- 每个期间资产 = 负债 + 所有者权益（主要检查）
- 现金余额与现金流量表期末现金一致
- 营运资本账户与支持性附表一致（如适用）
- 留存收益正确结转：期初留存收益 + 净利润 - 股息 +/- 调整 = 期末留存收益
- 债务余额与债务附表一致（如适用）
- 所有资产负债表项目具有适当的符号（资产为正数，大多数负债为正数）

**现金流量表（CF）质量检查**
- 经营活动现金流顶部的净利润与损益表净利润一致
- 非现金加回（折旧摊销、股份支付等）与其源附表/报表一致
- 营运资本变动符号正确（资产增加 = 现金使用 = 负数）
- 资本支出与固定资产附表或固定资产结转一致
- 筹资活动与资产负债表上债务和所有者权益账户的变动一致
- 期末现金与资产负债表现金一致
- 期初现金等于上一期间的期末现金

**支持性附表质量检查**
- 期初余额等于上一期间的期末余额
- 结转逻辑完整（期初 + 增加 - 减少 = 期末）
- 附表总计与主报表行项目一致
- 计算中使用的假设与假设工作表一致

### 步骤5：跨报表完整性检查

在验证各个工作表后，确认三张报表已正确整合：

| 检查 | 公式 | 预期结果 |
|------|------|----------|
| 资产负债表平衡 | 资产 - 负债 - 所有者权益 | = 0 |
| 现金对账 | 现金流量表期末现金 - 资产负债表现金 | = 0 |
| 净利润链接 | 损益表净利润 - 现金流量表起始净利润 | = 0 |
| 留存收益 | 期初留存收益 + 净利润 - 股息 - 资产负债表期末留存收益 | = 0（根据股份支付/其他项目进行调整） |

### 步骤6：最终审查

在认为模型完成之前：
- 遍历所有情景（如适用），验证在每种情况下检查均通过
- 审查所有 #REF!、#DIV/0!、#VALUE! 和 #NAME? 错误，进行解决或记录
- 确认所有输入单元格都已填充（搜索占位符值）
- 验证所有工作表的单位一致
- 在进行任何额外修改之前保存一个干净的版本

| 资产负债表平衡 | 资产 - 负债 - 权益 | = 0 |

| 月度与年度现金勾稽 | 期末现金（月度） - 期末现金（年度） | = 0 |
| 净利润关联 | 利润表净利润 - 现金流量表起始净利润 | = 0 |
| 留存收益 | 期初留存收益 + 净利润 + 股权激励费用 - 股利 - 资产负债表期末留存收益 | = 0 |
| 权益融资 | 普通股/额外实收资本变动（资产负债表） - 权益发行（筹资活动现金流） | = 0 |
| 第0年权益 | 募集权益（第0年） - 期初权益资本（第1年） | = 0 |

### 符号约定参考

| 报表 | 项目 | 符号约定 |
|-----------|------|-----------------|
| 经营活动现金流 | 折旧摊销、股权激励费用 | 正值（加回） |
| 经营活动现金流 | 应收账款变动（增加） | 负值（现金使用） |
| 经营活动现金流 | 应付账款变动（增加） | 正值（现金来源） |
| 投资活动现金流 | 资本支出 | 负值 |
| 筹资活动现金流 | 债务发行 | 正值 |
| 筹资活动现金流 | 债务偿还 | 负值 |
| 筹资活动现金流 | 股利 | 负值 |

### 循环引用处理

利息费用会产生循环引用：利息 → 净利润 → 现金 → 债务余额 → 利息

在Excel中启用迭代计算：文件 → 选项 → 公式 → 启用迭代计算。设置最大迭代次数为100，最大变化量为0.001。在假设标签页中添加循环中断开关。

### 检查类别

**第1节：货币一致性**
- 货币已在假设中识别并记录
- 所有标签页使用一致的货币符号和单位规模
- 单位行与模型货币匹配

**第2节：资产负债表完整性**
- 资产 = 负债 + 权益（每个期间）
- 公式：资产 - 负债 - 权益（必须 = 0）

**第3节：现金流量表完整性**
- 现金与资产负债表勾稽（现金流量表期末现金 = 资产负债表现金）
- 月度与年度现金勾稽：期末现金（月度） = 期末现金（年度）
- 净利润与利润表勾稽（现金流量表净利润 = 利润表净利润）
- 折旧摊销与折旧摊销表勾稽
- 股权激励费用与利润表勾稽
- 应收账款、存货、应付账款变动与营运资金表勾稽
- 资本支出与折旧摊销表勾稽

**第4节：留存收益**
- 留存收益滚动检查：期初留存收益 + 净利润 + 股权激励费用 - 股利 = 期末留存收益
- 显示各组成部分明细以便调试

**第5节：营运资金**
- 应收账款、存货、应付账款与资产负债表勾稽
- 应收账款周转天数、存货周转天数、应付账款周转天数合理性检查（超出正常范围时标记）

**第6节：债务明细表**
- 总债务与资产负债表勾稽（流动负债 + 长期债务）
- 利息计算与利润表勾稽

**第6b节：权益融资**
- 权益发行所得与资产负债表普通股/额外实收资本增加额勾稽
- 权益带来的现金增加 = 权益账户增加（必须平衡）
- 权益募集勾稽：普通股/额外实收资本变动（资产负债表） = 权益发行（筹资活动现金流）（必须 = 0）
- 第0年权益勾稽：募集权益（第0年） = 期初权益资本（第1年）

**第6c节：净经营亏损明细表**
- 期初净经营亏损（第1年 / 成立时） = 0（新企业从零净经营亏损开始）
- 仅当税前利润 &lt; 0时净经营亏损增加（亏损必须实现才能产生净经营亏损）
- 递延所得税资产与资产负债表勾稽（净经营亏损明细表递延所得税资产 = 资产负债表递延所得税资产）
- 净经营亏损使用 ≤ 税前利润的80%（2017年后联邦限制）
- 净经营亏损余额为非负数（不能超过可用余额）
- 仅当税前利润 &lt; 0时产生净经营亏损
- 当应税收入 ≤ 0时，所得税费用 = 0

**第7节：情景层级**
- 绝对指标：乐观 > 基准 > 悲观（净利润、息税折旧摊销前利润、自由现金流）
- 利润率：乐观 > 基准 > 悲观（毛利率%、息税折旧摊销前利润率%、净利润率%）
- 信用指标：杠杆率方面乐观 &lt; 基准 &lt; 悲观（反向关系）

**第8节：公式完整性**
- 销售成本、销售及管理费用、研发费用、股权激励费用由收入百分比驱动（无硬编码）
- 预测年度之间公式一致
- 无 #REF!、#DIV/0!、#VALUE! 错误

**第9节：信用指标阈值**
- 根据契约阈值将指标标记为绿/黄/红
- 任何红旗汇总

### 主检查公式

将所有部分状态汇总为单一主检查：
- 如果所有部分通过 → "✓ 所有检查通过"
- 如果任何部分失败 → "✗ 检测到错误 - 请查阅以下内容"

### 快速调试工作流程

当主状态显示错误时：
1. 滚动查找红色高亮部分
2. 确定哪个检查类别存在失败项
3. 导航到源标签页进行调查
4. 修复底层问题
5. 返回检查标签页验证解决情况
