# 可比公司分析

## ⚠️ 关键：数据源优先级（首先阅读）

**必须始终遵循此数据源层级：**

1.  **第一：检查 MCP 数据源** - 如果有 S&P Kensho MCP、FactSet MCP 或 Daloopa MCP 可用，请仅使用它们获取财务和交易信息
2.  **如果上述 MCP 数据源可用，不要使用网络搜索**
3.  **仅在 MCP 不可用时：** 则使用彭博终端、SEC EDGAR 文件或其他机构来源
4.  **绝不要将网络搜索作为主要数据源** - 它缺乏机构级分析所需的准确性、审计线索和可靠性

**为什么这很重要：** MCP 来源提供经过验证、具有机构级质量并带有适当引用的数据。网络搜索结果可能过时、不准确或对金融分析不可靠。

## 概述
此技能教导智能体构建结合运营指标、估值倍数和统计基准分析的机构级可比公司分析。输出是一个结构化的 Excel/电子表格，通过同行比较支持明智的投资决策。

**参考资料与情境化：**

`examples/comps_example.xlsx` 中提供了一个可比公司分析示例。当在此技能目录中使用此示例文件或其他示例文件时，请智能地使用它们：

**可以将示例用于：**
- 理解结构层次（各部分如何流动）
- 把握所要求的严格程度（统计深度、文档标准）
- 学习原则（清晰的标题、透明的公式、审计线索）

**不要将示例用于：**
- 精确复制格式或指标
- 不考虑上下文地复制布局
- 不考虑受众就应用相同的视觉风格

**始终先问自己：**
1.  **“您有偏好的格式，还是我应该采用模板风格？”**
2.  **“受众是谁？”**（投资委员会、董事会演示、快速参考、详细备忘录）
3.  **“关键问题是什么？”**（估值、增长分析、竞争定位、效率）
4.  **“上下文是什么？”**（并购评估、投资决策、行业基准、绩效评估）

**根据具体情况调整：**
- **行业背景**：大型科技巨头需要的指标与新兴 SaaS 初创公司不同
- **行业特定需求**：尽早添加相关指标（例如，云年度经常性收入、企业客户、开发者生态对于科技行业）
- **公司熟悉度**：知名公司可能需要更少背景介绍，更多关注变化分析
- **决策类型**：并购与持续的组合监控所需的侧重点不同

**核心原则**：使用模板原则（清晰的结构、统计严谨性、透明的公式），但根据上下文改变执行方式。目标是机构级质量的分析，而不是看起来像机构模板的分析。

用户提供的示例和明确的偏好始终优先于默认设置。

## 核心哲学
**“先建立正确的结构，然后让数据讲述故事。”**

从强制思考什么最重要的标题开始，输入干净的数据，构建透明的公式，并让统计数据自动呈现。一个好的可比分析应该让构建它的人之外的人也能立即理解。

---

## ⚠️ 关键：公式优于硬编码 + 逐步验证

**公式，而非硬编码：**
- 每个派生值（利润率、倍数、统计量）必须是引用输入单元格的 Excel 公式 — 绝不是粘贴的预计算数字
- 使用 Python/openpyxl 构建工作表时：写 `cell.value = "=E7/C7"` (公式字符串)，而不是 `cell.value = 0.687` (计算结果)
- 唯一可以硬编码的值应该是原始输入数据（收入、EBITDA、股价等）— 并且每一个都要在单元格注释中注明来源
- 原因：当输入更改时，模型必须自动更新。硬编码的利润率是等待发生的沉默错误。

**与用户逐步验证：**
- 设置好结构后 → 向用户展示标题布局，然后再填充数据
- 输入原始输入数据后 → 向用户展示输入块，并在构建公式前确认来源/时间范围
- 构建运营指标公式后 → 展示计算出的利润率，并在转到估值前与用户进行合理性检查
- 构建估值倍数后 → 展示倍数，并在添加统计量前确认它们看起来合理
- 不要从头到尾构建整个工作表然后再呈现 — 通过确认每个部分尽早发现错误

## 第一部分：文档结构与设置

### 标题块（第 1-3 行）
```
第 1 行：[分析标题] - 可比公司分析
第 2 行：[公司列表及其股票代码] • [公司 1 (TICK1)] • [公司 2 (TICK2)] • [公司 3 (TICK3)]
第 3 行：截至 [期间] | 除每股金额和比率外，所有数字单位均为 [百万美元/十亿美元]
```

**为什么这很重要：** 立即建立上下文。任何打开此文件的人都能知道他们在看什么，它是何时创建的，以及如何解读这些数字。

### 视觉规范标准（可选 - 用户偏好和上传的模板始终优先）

**重要：这些仅为建议的默认设置。始终优先考虑：**
1. 用户明确的格式偏好
2. 任何上传模板文件的格式
3. 公司/团队风格指南
4. 这些默认设置（仅在没有其他指导提供时）

**建议的字体与排版：**
- **字体族**：Times New Roman（专业、易读、行业标准）
- **字号**：数据单元格 11pt，标题 12pt
- **粗体文本**：章节标题、公司名称、统计标签

**默认配色与底纹 — 专业蓝/灰色调（越少越好）：**
- **保持克制** — 只使用蓝色和灰色。不要引入绿色、橙色、红色或多种强调色。一个干净的可比分析表总共使用 3-4 种颜色。
- **章节标题**（例如，“运营统计与财务指标”）：
  - 深蓝色背景（`#1F4E79` 或 `#17365D` 海军蓝）
  - 白色粗体文本
  - 所有列全行底纹
- **列标题**（例如，“公司”、“收入”、“利润率”）：
  - 浅蓝色背景（`#D9E1F2` 或类似的淡蓝色）
  - 黑色粗体文本
  - 居中对齐
- **数据行**：
  - 公司数据白色背景
  - 公式使用黑色文本；硬编码输入使用蓝色文本
- **统计行**（最大值、第 75 百分位数等）：
  - 浅灰色背景（`#F2F2F2`）
  - 黑色文本，左对齐标签
- **这就是全部配色**：深蓝 + 浅蓝 + 浅灰 + 白色。除非用户的模板另有说明，否则不使用其他颜色。

**建议的格式规范：**
- **小数精度**：
  - 百分比：1 位小数 (12.3%)
  - 倍数：1 位小数 (13.5x)
  - 美元金额：无小数，千位分隔符 (69,632)
  - 利润率显示为百分比：1 位小数 (68.7%)
- **边框**：无边框（干净、极简外观）
- **对齐**：所有指标居中对齐，以实现干净、统一的外观
- **单元格尺寸**：所有列宽应统一/均匀，所有行高应一致（创建干净、专业的网格）

**注意：** 如果用户提供了模板文件或指定了不同的格式，则改用该格式。

## 第二部分：运营统计与财务指标

### 核心列（从这些开始）
1.  **公司** - 格式一致的名称
2.  **收入** - 规模指标（可以是过去十二个月 LTM、季度或年度，取决于上下文）
3.  **收入增长** - 同比百分比变化
4.  **毛利润** - 收入减去销售成本
5.  **毛利率** - 毛利润/收入（基本盈利能力）
6.  **EBITDA** - 息税折旧摊销前利润
7.  **EBITDA 利润率** - EBITDA/收入（运营效率）

### 可选项（根据行业/目的选择）
- **季度对比 LTM** - 如果季节性重要，则包含两者
- **自由现金流** - 对于资本密集型或 SaaS 企业
- **自由现金流利润率** - 自由现金流/收入（现金生成效率）
- **净利润** - 对于成熟、盈利的公司
- **营业利润** - 对于折旧摊销差异较大的业务
- **资本支出指标** - 对于重资产行业
- **40 法则** - 专用于 SaaS（增长 % + 利润率 %）
- **现金流转换率** - 用于盈利质量分析（高级）

### 公式示例（使用第 7 行为例）
```excel
// 核心比率 - 这些总是被计算
毛利率 (F7): =E7/C7
EBITDA 利润率 (H7): =G7/C7

// 可选比率 - 如果相关则包括
自由现金流利润率: =[自由现金流]/[收入]
净利润率: =[净利润]/[收入]
40 法则: =[增长 %]+[自由现金流利润率 %]
```

**黄金法则：** 每个比率都应该是 [某指标] / [收入] 或 [某指标] / [本表中的某指标]。保持简单。

### 统计块（在公司数据之后）

**关键：为所有可比指标（比率、利润率、增长率、倍数）添加统计公式。**

```
[留一个空行用于视觉分隔]
- 最大值：=MAX(B7:B9)
- 第 75 百分位数：=QUARTILE(B7:B9,3)
- 中位数：=MEDIAN(B7:B9)
- 第 25 百分位数：=QUARTILE(B7:B9,1)
- 最小值：=MIN(B7:B9)
```

**需要统计的列（可比指标）：**
- 收入增长 %、毛利率 %、EBITDA 利润率 %、每股收益
- 企业价值/收入、企业价值/EBITDA、市盈率、股息收益率 %、贝塔系数

**不需要统计的列（规模指标）：**
- 收入、EBITDA、净利润（绝对规模因公司规模而异）
- 市值、企业价值（在不同规模的公司之间不可比）

**注意：** 在公司数据行和统计行之间添加一个空行用于视觉分隔。不要添加“行业统计”或“估值统计”标题行。

**为什么四分位数很重要：** 它们展示的是分布，而不仅仅是平均值。第 75 百分位数的倍数告诉你“溢价”公司的交易水平。

## 第3部分：估值倍数与投资指标

### 核心估值列（从这些开始）
1.  **公司** - 与运营部分相同的顺序
2.  **市值** - 当前市场估值
3.  **企业价值** - 市值 ± 净债务/现金
4.  **EV/收入（企业价值/收入）** - 市场为每美元销售额支付多少
5.  **EV/EBITDA（企业价值/EBITDA）** - 市场为每美元盈利支付多少
6.  **市盈率** - 价格相对于净收益的比率

### 可选估值指标（根据上下文选择）
- **自由现金流收益率** - 自由现金流/市值（用于关注现金的分析）
- **PEG比率** - 市盈率/增长率（适用于成长型公司）
- **市净率** - 市值相对于账面价值（适用于重资产企业）
- **ROE/ROA（净资产收益率/总资产收益率）** - 回报指标（用于盈利能力比较）
- **收入/EBITDA复合年增长率** - 历史增长率（用于趋势分析）
- **资产周转率** - 收入/资产（用于运营效率分析）
- **债务/权益** - 杠杆率（用于资本结构分析）

**核心原则：** 包含3-5个对你所在行业重要的核心倍数。不要仅仅因为你可以就包含每一个可能的指标。

### 公式示例
```excel
// 核心倍数 - 始终包含这些
EV/收入: =[企业价值]/[LTM 收入]
EV/EBITDA: =[企业价值]/[LTM EBITDA]
市盈率: =[市值]/[净利润]

// 可选倍数 - 如果数据可用则包含
自由现金流收益率: =[LTM 自由现金流]/[市值]
PEG比率: =[市盈率]/[增长率 %]
```

### 交叉引用规则
**关键：** 估值倍数必须引用运营指标部分的数据。永远不要两次输入相同的原始数据。如果收入在C7单元格，那么EV/收入的公式应该引用C7。

### 统计模块
结构与运营部分相同：对每个指标计算最大值、第75百分位数、中位数、第25百分位数、最小值。在公司数据和统计信息之间添加一个空行以进行视觉分隔。**不要**添加"估值统计"标题行。

## 第4部分：注释与方法论文档

### 必需组成部分

**数据来源与质量：**
- 数据来自何处？（S&P Kensho MCP, FactSet MCP, Daloopa MCP, Bloomberg, SEC文件）
- 它覆盖哪个时间段？（2024年第四季度，经审计的数据）
- 如何验证？（与10-K/10-Q报告交叉核对）
- 注意：优先使用MCP数据源（S&P Kensho, FactSet, Daloopa）以获得更好的准确性和可追溯性。

**关键定义：**
- EBITDA的计算方法（毛利润 + 折旧摊销，或营业收入 + 折旧摊销）
- 自由现金流公式（经营现金流 - 资本支出）
- 特殊指标解释（Rule of 40, FCF转化率）
- 时间段定义（LTM，CAGR计算期间）

**估值方法论：**
- 企业价值如何计算？（市值 + 净债务）
- 使用了哪些增长率？（历史CAGR，前瞻性估计）
- 是否进行了任何调整？（排除一次性项目，标准化利润率）

**分析框架：**
- 投资论点是什么？（云/软件即服务效率）
- 哪些指标最重要？（现金生成，资本效率）
- 读者应如何解读统计数据？（四分位数提供上下文）

## 第五部分：选择正确的指标（决策框架）

### 从“我在回答什么问题？”开始

**“哪家公司被低估了？”**
→ 关注：企业价值/收入、企业价值/息税折旧摊销前利润、市盈率、市值
→ 忽略：运营细节、增长指标

**“哪家公司效率最高？”**
→ 关注：毛利率、EBITDA利润率、自由现金流利润率、资产周转率
→ 忽略：规模指标、绝对美元金额

**“哪家公司增长最快？”**
→ 关注：收入增长率 %、EBITDA复合年增长率、用户/客户增长
→ 忽略：利润率指标、杠杆比率

**“哪家公司是最佳的现金生成器？”**
→ 关注：自由现金流、自由现金流利润率、自由现金流转化率、资本支出强度
→ 忽略：EBITDA、市盈率

### 行业特定指标选择

**软件/SaaS：**
必备：收入增长、毛利率、Rule of 40
可选：年度经常性收入、净收入留存率、客户获取成本回收期
忽略：资产周转率、库存指标

**制造业/工业：**
必备：EBITDA利润率、资产周转率、资本支出/收入
可选：资产回报率、库存周转率、订单积压
忽略：Rule of 40、SaaS指标

**金融服务：**
必备：净资产收益率、资产回报率、效率比率、市盈率
可选：净息差、贷款损失准备金
忽略：毛利率、EBITDA（对银行意义不大）

**零售/电子商务：**
必备：收入增长、毛利率、库存周转率
可选：同店销售额、客户获取成本
忽略：研发或资本支出指标过重

### “5-10 法则”

**5 个运营指标** - 收入、增长、2-3 个利润率/效率指标
**5 个估值指标** - 市值、企业价值、3 个倍数
**= 共 10 列** - 足以讲述完整故事，又不会太多导致主线不清

如果你的指标超过 15 个，很可能包含了噪音。请大刀阔斧地删减。

## 第六部分：最佳实践与质量检查

### 开始之前
1. **定义可比公司组** - 公司必须真正可比（相似的商业模式、规模、地理区域）
2. **选择合适的时期** - LTM 平滑季节性；季度显示趋势
3. **提前标准化单位** - 以百万还是十亿为单位，这个决定会影响一切
4. **映射数据来源** - 明确每个数字的出处

### 构建过程中
1. **首先输入所有原始数据** - 在编写公式之前完成蓝色文字部分
2. **为所有硬编码输入添加单元格注释** - 右键单击单元格 → 插入批注 → 记录来源或假设

   **对于有来源的数据，精确引用其出处：**
   - 示例：“Bloomberg终端 - MSFT Equity DES，访问于2024-10-02”
   - 示例：“2024年第四季度10-K文件，第42页，项目‘总收入’”
   - 示例：“FactSet共识估计，截至2024-10-02”
   - **尽可能包含超链接**：右键单击单元格 → 链接 → 粘贴SEC文件、数据源或报告的URL

   **对于假设，解释推理过程：**
   - 示例：“基于同行中位数假设15%的EBITDA利润率，公司未披露”
   - 示例：“估计企业价值为市值 + 5000万美元净债务（来自第三季度资产负债表，第四季度尚未公布）”
   - 示例：“基于华尔街共识每股收益3.45美元的远期市盈率（12位分析师估计的平均值）”

   **为何重要**：支持审计跟踪、数据验证、假设透明化和未来更新
3. **逐行构建公式** - 在继续之前测试每个计算
4. **为标题使用绝对引用** - $C$6 锁定标题行
5. **格式保持一致** - 百分比显示为百分比，而非小数
6. **添加条件格式** - 自动突出显示异常值

### 合理性检查
- **利润率测试**：毛利率 > EBITDA利润率 > 净利润率（根据定义，此关系始终成立）
- **倍数合理性**：
  - 企业价值/收入：通常0.5-20倍（因行业差异很大）
  - 企业价值/EBITDA：通常8-25倍（跨行业相当一致）
  - 市盈率：通常10-50倍（取决于增长率）
- **增长-倍数相关性**：更高的增长通常意味着更高的倍数
- **规模-效率权衡**：大型公司通常有更好的利润率（规模效益）

### 应避免的常见错误
❌ 在公式中混用市值和企业价值
❌ 分子和分母使用不同的时间段（LTM 对 季度）
❌ 将数字硬编码到公式中，而不是使用单元格引用
❌ **硬编码的输入单元格没有注释引用来源或解释假设**
❌ 在可用时缺少指向SEC文件或数据源的超链接
❌ 包含太多没有明确目的的指标
❌ 包含不可比的公司（不同的商业模式）
❌ 使用过时数据而未加说明
❌ 错误计算百分比的平均值（应使用中位数）

## 第七部分：工作流程与实用技巧

### 分步流程
1. **搭建结构**（30分钟）
   - 创建所有标题
   - 格式化单元格（蓝色表示输入，黑色表示公式）
   - 确定单位和日期参考

2. **收集数据**（60-90分钟）
   - 从主要来源获取（S&P Kensho MCP、FactSet MCP、Daloopa MCP（如可用）；否则使用Bloomberg、SEC）
   - 在蓝色区域输入所有原始数字
   - 在注释部分记录来源

3. **构建公式**（30分钟）
   - 从简单比率开始（利润率）
   - 进展到倍数（企业价值/收入）
   - 添加交叉检查（利润率是否合理？）

4. **添加统计**（15分钟）
   - 为所有列复制公式结构
   - 验证范围是否正确（B7:B9，而不是B7:B10）
   - 检查四分位逻辑

5. **质量控制**（30分钟）
   - 运行合理性检查
   - 验证公式引用
   - 检查 #DIV/0! 或 #REF! 错误
   - 与已知基准进行比较

6. **文档化**（15分钟）
   - 完成注释部分
   - 添加数据来源
   - 定义方法论
   - 给分析加上日期戳

## 第十节：危险信号与警告

### 数据质量问题
🚩 时间段不一致（混合使用季度和年度数据）
🚩 缺失数据且未作说明
🚩 数据源之间存在显著差异（方差>10%）

### 估值危险信号
🚩 对EBITDA为负的公司使用EBITDA倍数估值（应改用营收倍数）
🚩 市盈率>100倍且没有超高速增长故事
🚩 毛利率对该行业而言不合理

### 可比性问题
🚩 财年结束时间不同（导致时间错配）
🚩 混合了纯业务公司和综合性集团
🚩 业务模式存在重大差异却被标记为“可比公司”

**存疑时，排除该公司。** 拥有3个完美的可比公司优于6个有问题的公司。

## 第十一节：公式参考指南

### 基本Excel公式
```excel
// 统计函数
=AVERAGE(范围)          // 简单平均值
=MEDIAN(范围)           // 中位数
=QUARTILE(范围, 1)      // 第25百分位数
=QUARTILE(范围, 3)      // 第75百分位数
=MAX(范围)              // 最大值
=MIN(范围)              // 最小值
=STDEV.P(范围)          // 标准差

// 财务计算
=B7/C7                   // 简单比率（利润率）
=SUM(B7:B9)/3            // 多家公司的平均值
=IF(B7>0, C7/B7, "N/A")  // 条件计算
=IFERROR(C7/D7, 0)       // 处理除以零错误

// 跨工作表引用
='Sheet1'!B7             // 引用另一个工作表
=VLOOKUP(A7, 表1, 2)     // 从数据表中查找
=INDEX(MATCH())          // 高级查找

// 格式设置
=TEXT(B7, "0.0%")        // 格式化为百分比
=TEXT(C7, "#,##0")       // 千位分隔符
```

### 常用比率公式
```excel
毛利率 = 毛利润 / 营收
EBITDA利润率 = EBITDA / 营收
自由现金流利润率 = 自由现金流 / 营收
自由现金流转化率 = 自由现金流 / 经营现金流
净资产收益率 = 净利润 / 股东权益
总资产收益率 = 净利润 / 总资产
资产周转率 = 营收 / 总资产
债务权益比 = 总负债 / 股东权益
```

## 关键原则总结

1.  **结构驱动洞察** - 正确的标题引导正确的思考
2.  **少即是多** - 5-10个关键指标胜过20个不相关的指标
3.  **根据问题选择指标** - 估值分析 ≠ 效率分析
4.  **统计揭示模式** - 中位数/四分位数比平均值更能说明问题
5.  **透明优于复杂** - 每个人都能理解的简单公式
6.  **可比性为王** - 宁可排除也不要强行使用差的可比公司
7.  **记录您的选择** - 在注释部分解释选择了哪些指标及原因
