# 并购模型

为并购交易构建增厚/稀释分析。模拟摊薄后每股收益影响、协同效应敏感性和购买价格分配。在评估潜在收购、为推介准备并购后果分析或就交易条款提供建议时使用。

## 工作流程

### 步骤 1：收集输入

**收购方：**
- 公司名称、当前股价、流通股数
- LTM 和 NTM 每股收益（GAAP 和调整后）
- 市盈率倍数
- 税前债务成本、税率
- 资产负债表上的现金、现有债务

**目标方：**
- 公司名称、当前股价、流通股数（如果上市）
- LTM 和 NTM 每股收益或净收入
- 企业价值或股权价值

**交易条款：**
- 每股收购报价（或相对于当前股价的溢价）
- 对价组合：% 现金 vs. % 股票
- 为现金部分筹集的新债务
- 预期协同效应（收入和成本）及分阶段实施时间表
- 交易费用和融资成本
- 预期完成日期

## 重要提示

- 在相关处始终同时显示 GAAP 和调整后（现金）每股收益
- 股票交易：使用收购方当前股价计算交换比率，注意新股发行导致的稀释
- 包含购买价格分配 — 商誉和无形资产摊销对 GAAP 每股收益很重要
- 协同效应的分阶段实施至关重要 — 第 1 年通常仅为稳定状态协同效应的 25-50%
- 不要忘记放弃的现金利息收入和新债产生的利息支出
- 协同效应和利息调整的税率应与收购方的边际税率匹配


## 数据来源 — MCP 优先，网络备选

下文多处提到“使用 S&P Kensho MCP / Daloopa MCP / FactSet MCP”。这些是来自原始 Cowork 插件上下文的商业金融数据 MCP。在 Hermes 中：

- **如果你配置了任何结构化金融数据 MCP**（Hermes 支持 MCP — 见 `native-mcp` 技能），优先使用它获取时点可比公司、先例交易和申报文件。
- **否则**，回退到：
  - `web_search` / `web_extract` 访问 SEC EDGAR (`https://www.sec.gov/cgi-bin/browse-edgar`) 获取美国申报文件
  - 公司投资者关系页面获取新闻稿、收益演示文稿
  - `browser_navigate` 用于交互式数据门户
  - 用户提供的数据（当上下文中没有时，明确询问）
- **切勿捏造**。如果无法获取某个倍数、先例或申报文件编号，将单元格标记为 `[UNSOURCED]` 并提请用户注意。
