# DCF 模型构建器

## 概述

此技能按照投资银行标准创建用于股权估值的机构级 DCF 模型。每次分析都会生成一个详细的 Excel 模型（敏感性分析包含在 DCF 工作表的底部）。

## 工具

- 默认使用用户提供的所有信息以及可用于数据获取的 MCP 服务器。

## 关键约束 - 请先阅读

这些约束适用于所有 DCF 模型构建。开始前请先复习：

**公式优于硬编码（不可协商）：**
- 每个预测、利润率、折现因子、现值和敏感性单元格**必须**是活动的 Excel 公式 —— 绝不能是 Python 计算后写入的数字值
- 使用 openpyxl 时：`ws["D20"] = "=D19*(1+$B$8)"` 是正确的；`ws["D20"] = calculated_revenue` 是**错误**的
- 唯一允许的硬编码数字是：(1) 原始历史输入，(2) 假设驱动因素（增长率、WACC 输入、永续增长率 g），(3) 当前市场数据（股价、债务余额）
- 如果你发现自己正在用 Python 计算某个值并将其写入 —— **停止**。模型必须能够在用户更改假设时动态调整。

**与用户逐步验证（不要端到端构建）：**
- 数据检索后 → 向用户展示原始输入块（收入、利润率、股份、净债务）并在预测前确认
- 收入预测后 → 展示预测的营收总额和增长率，在构建利润构建前确认
- 自由现金流构建后 → 展示完整的自由现金流计划，在计算 WACC 前确认逻辑
- WACC 后 → 展示计算和输入，在折现前确认
- 终值 + 现值后 → 展示股权桥梁（企业价值 → 股权价值 → 每股价值），在构建敏感性表前确认
- 在每个阶段捕获错误 —— 在敏感性表构建完成后才发现一个错误的利润假设意味着需要重建其下游的所有内容

**敏感性表：**
- **使用奇数行和列**（标准：5×5，有时 7×7）—— 这保证了有一个真正的中心单元格
- **中心单元格 = 基准情景。** 构建轴值，使得中间行标题和中间列标题**精确等于**模型的实际假设（例如，如果基准 WACC = 9.0%，则中间行是 9.0%；如果永续增长率 g = 3.0%，则中间列是 3.0%）。因此，中心单元格的输出必须等于模型的实际隐含股价 —— 这是验证表格构建正确的完整性检查。
- 使用中等蓝色填充（`#BDD7EE`）+ 粗体字**高亮显示中心单元格**，以便立即识别哪个单元格是基准情景。
- 用完整的 DCF 重新计算公式填充**所有**单元格（通常 3 个表 × 25 个单元格 = 75 个）
- 使用 openpyxl 循环以编程方式写入公式
- **禁止**占位符文本，**禁止**线性近似，**禁止**需要手动操作的步骤
- 每个单元格必须为该特定假设组合重新计算完整的 DCF

**单元格批注：**
- 在创建每个硬编码值**时**就添加单元格批注
- 格式：“来源：[系统/文档]，[日期]，[引用]，[如果适用，提供网址]”
- 每个蓝色输入单元格在进入下一部分之前都必须有批注
- 不要推迟到最后或写上“待办：添加来源”

**模型布局规划：**
- 在编写任何公式**之前**，定义**所有**部分的行位置
- 首先写入**所有**标题和标签
- 其次写入**所有**分隔符和空白行
- **然后**使用锁定的行位置编写公式
- 在创建后立即测试公式

**公式重新计算：**
- 在交付前运行 `python recalc.py model.xlsx 30`
- 修复**所有**错误直到状态显示“成功”
- 要求零公式错误（#REF!、#DIV/0!、#VALUE! 等）

**情景块：**
- 为熊市/基准/牛市情景创建单独的块
- 在每个块中，跨预测年份水平展示假设
- 使用 IF 公式：`=IF($B$6=1,[熊市单元格],IF($B$6=2,[基准单元格],[牛市单元格]))`
- 验证公式是否引用了正确的情景块单元格

## DCF 流程工作流

### 步骤 1：数据检索与验证

从 MCP 服务器、用户提供的数据和网络获取数据。

**数据来源优先级：**
1.  **MCP 服务器**（如果已配置）- 来自 Daloopa 等提供商的结构化财务数据
2.  **用户提供的数据** - 来自他们研究的历史财务数据
3.  **网络搜索/获取** - 当前价格、贝塔值、债务和现金（如需要）

**验证清单：**
- 核实净债务 vs 净现金（对估值至关重要）
- 确认摊薄后流通股（检查最近的回购/发行）
- 验证历史利润率是否与商业模式一致
- 交叉核对收入增长率与行业基准
- 验证税率是否合理（通常为 21-28%）

### 步骤 2：历史分析（3-5 年）

分析并记录：
- **收入增长趋势**：计算复合年增长率，识别驱动因素
- **利润走势**：跟踪毛利率、营业利润率、自由现金流利润率
- **资本密集度**：折旧摊销和资本支出占收入的百分比
- **营运资本效率**：净营运资本变化占收入增长的百分比
- **回报指标**：投资资本回报率、股本回报率趋势

创建摘要表显示：
```
历史指标（最近十二个月）：
收入：X 百万美元
收入增长：X% 复合年增长率
毛利率：X%
营业利润率：X%
折旧摊销占收入比：X%
资本支出占收入比：X%
自由现金流利润率：X%
```

### 步骤 3：构建收入预测

**方法论：**
1.  从最新的实际收入（最近十二个月或最近财年）开始
2.  对每个预测年份应用增长率
3.  同时显示金额和计算出的增长百分比

**增长率框架：**
- 第 1-2 年：反映近期可见性的较高增长
- 第 3-4 年：逐渐向行业平均水平靠拢
- 第 5 年及以后：接近永续增长率

**公式结构：**
- 收入（第 N 年）= 收入（第 N-1 年）× (1 + 增长率)
- 增长率（第 N 年）= 收入（第 N 年）/ 收入（第 N-1 年） - 1

**三情景方法：**
```
熊市情景：保守增长（例如，8-12%）
基准情景：最可能的场景（例如，12-16%）
牛市情景：乐观增长（例如，16-20%）
```

### 步骤 4：运营费用建模

**固定/可变成本分析：**

运营费用应建模为现实的经营杠杆：
- **销售与营销**：通常占收入的 15-40%，取决于商业模式
- **研发**：对于科技公司，通常占 10-30%
- **一般与行政**：通常占收入的 8-15%，随着公司规模扩大显示出杠杆效应

**关键原则：**
- **所有**百分比都基于**收入**，而非毛利
- 建模经营杠杆：随着收入规模扩大，百分比应下降
- 为销售营销、研发、一般行政设置单独的行项目
- 计算营业利润 = 毛利 - 总运营费用

**利润扩张框架：**
```
当前状态 → 目标状态（第 5 年）
毛利率：X% → Y%（基于规模、效率证明合理性）
营业利润率：X% → Y%（收入增长 + 运营杠杆的结果）
```

### 步骤 5：自由现金流计算

**按正确顺序构建自由现金流：**

```
营业利润
(-) 税费（营业利润 × 税率）
= 税后净营业利润
(+) 折旧摊销（非现金费用，占收入的百分比）
(-) 资本支出（占收入的百分比，通常为 4-8%）
(-) 净营运资本变动
= 无杠杆自由现金流
```

**营运资本建模：**
- 按收入变化（收入增量）的百分比计算
- 典型范围：收入变化的 -2% 至 +2%
- 负数 = 现金来源（营运资本释放）
- 正数 = 现金使用（营运资本积累）

**维护性 vs 增长性资本支出：**
- 维护性资本支出：维持当前运营（约占收入 2-3%）
- 增长性资本支出：支持扩张（额外占收入 2-5%）
- 总资本支出应与公司的增长战略保持一致

### 步骤 6：资本成本（WACC）研究

**资本资产定价模型用于计算权益成本：**

```
权益成本 = 无风险利率 + 贝塔值 × 股权风险溢价

其中：
- 无风险利率 = 当前 10 年期国债收益率
- 贝塔值 = 5 年期月度股票贝塔值 vs 市场指数
- 股权风险溢价 = 5.0-6.0%（市场标准）
```

**债务成本计算：**

```
税后债务成本 = 税前债务成本 × (1 - 税率)

通过以下方式确定税前债务成本：
- 信用评级（如果可用）
- 公司债券的当前收益率
- 财务报表中的利息费用 / 总债务
```

**资本结构权重：**

```
股权市场价值 = 当前股价 × 流通股数
净债务 = 总债务 - 现金及等价物
企业价值 = 市值 + 净债务

股权权重 = 市值 / 企业价值
债务权重 = 净债务 / 企业价值

WACC = (权益成本 × 股权权重) + (税后债务成本 × 债务权重)
```

**特殊情况：**
- **净现金头寸**：如果现金 > 债务，则净债务为**负数**
  - 债务权重可能为负
  - WACC 计算相应调整
- **无债务**：WACC = 权益成本

**典型 WACC 范围：**
- 大型股，稳定：7-9%
- 成长型公司：9-12%
- 高增长/高风险：12-15%

### 步骤 7：折现率应用（5-10 年预测期）

**年中惯例：**
- 假设现金流在年中发生
- 折现期间：0.5, 1.5, 2.5, 3.5, 4.5 等
- 折现因子 = 1 / (1 + WACC)^期间

**现值计算：**
```
对于每个预测年份：
自由现金流的现值 = 无杠杆自由现金流 × 折现因子

示例（第 1 年）：
自由现金流 = $1,000
WACC = 10%
期间 = 0.5
折现因子 = 1 / (1.10)^0.5 = 0.9535
现值 = $1,000 × 0.9535 = $954
```

**预测期选择：**
- **5 年**：大多数分析的标准
- **7-10 年**：具有较长增长跑道的高增长公司
- **3 年**：成熟、稳定的业务

### 步骤 8：终值计算

**永续增长法（首选）：**

```
终年自由现金流 = 最终年自由现金流 × (1 + 永续增长率)
终值 = 终年自由现金流 / (WACC - 永续增长率)

关键约束：永续增长率 < WACC（否则价值为无穷大）
```

**永续增长率选择：**
- 保守：2.0-2.5%（GDP 增长率）
- 适中：2.5-3.5%
- 激进：3.5-5.0%（仅适用于市场领导者）

**不要超过**：无风险利率或长期 GDP 增长率

**退出乘数法（替代方法）：**
```
终值 = 最终年息税折旧摊销前利润 × 退出乘数

其中退出乘数来自：
- 行业可比交易乘数
- 先例交易乘数
- 典型范围：8-15 倍息税折旧摊销前利润
```

**终值的现值：**
```
终值的现值 = 终值 / (1 + WACC)^最终期间

其中最终期间考虑了时间因素：
采用年中惯例的 5 年模型：期间 = 4.5
```

**终值完整性检查：**
- 应占企业价值的 50-70%
- 如果 >75%，模型可能过度依赖终值假设
- 如果 &lt;40%，检查终值假设是否过于保守

### 步骤 9：企业价值到股权价值桥梁

**估值摘要结构：**

```
(+) 预测期自由现金流的现值之和 = $X 百万
(+) 终值的现值 = $Y 百万
= 企业价值 = $Z 百万

(-) 净债务 [或如果为负则 + 净现金] = $A 百万
= 股权价值 = $B 百万

÷ 摊薄后流通股数 = C 百万股
= 隐含每股股价 = $XX.XX

当前股价 = $YY.YY
隐含回报率 = (隐含股价 / 当前股价) - 1 = XX%
```

**关键调整：**
- **净债务 = 总债务 - 现金及等价物**
  - 如果为正：从企业价值中扣除（减少股权价值）
  - 如果为负（净现金）：加入企业价值（增加股权价值）
- **使用摊薄后流通股**：包括期权、限制性股票单位、可转换证券
- **其他调整**（如适用）：
  - 少数股东权益
  - 养老金负债
  - 经营租赁义务

**估值输出格式：**
```csv
估值组成部分，金额（$M）
预测期自由现金流现值，X.X
终值现值，Y.Y
企业价值，Z.Z
(-) 净债务，A.A
股权价值，B.B
,,
流通股数（百万），C.C
隐含每股股价，$XX.XX
当前股价，$YY.YY
隐含上涨/(下跌)空间，+XX%
```

### 步骤 10：敏感性分析

在 DCF 工作表底部构建**三个敏感性表**，显示估值如何随不同假设变化：

1.  **WACC vs 永续增长率** - 显示企业价值对折现率和永续增长的敏感性
2.  **收入增长率 vs 营业利润率** - 显示营收增长和经营杠杆的影响
3.  **贝塔值 vs 无风险利率** - 显示对权益成本组成部分的敏感性

**实现方式**：这些是简单的二维网格（**不是** Excel 的“数据表”功能），每个单元格都包含公式。每个单元格必须包含针对该特定假设组合的完整 DCF 重新计算。有关使用 openpyxl 以编程方式填充所有 75 个单元格的详细要求，请参阅“关键约束”部分。

### 情景块选择模式 - 遵循此方法

**假设按情景组织在单独的块中：**

**关键结构 - 每个部分标题三行：**

```csv
熊市情景假设（部分标题，跨列合并单元格）
假设，FY1，FY2，FY3，FY4，FY5
收入增长率（%），12%，10%，9%，8%，7%
营业利润率（%），45%，44%，43%，42%，41%

基准情景假设（部分标题，跨列合并单元格）
假设，FY1，FY2，FY3，FY4，FY5
收入增长率（%），16%，14%，12%，10%，9%
营业利润率（%），48%，49%，50%，51%，52%

牛市情景假设（部分标题，跨列合并单元格）
假设，FY1，FY2，FY3，FY4，FY5
收入增长率（%），20%，18%，15%，13%，11%
营业利润率（%），50%，51%，52%，53%，54%
```

**每个情景块必须有一个列标题行**，显示预测年份（FY2025E, FY2026E 等），位于部分标题正下方。没有这个，用户无法判断哪个假设值对应哪个年份。

**如何引用假设 - 创建一个整合列：**
1.  情景选择器单元格（例如 B6）包含 1=熊市，2=基准，或 3=牛市
2.  创建一个整合列，使用 INDEX 或 OFFSET 公式从正确的情景块中提取数据
3.  预测公式引用该整合列（干净的单元格引用）
4.  每个情景块包含跨预测年份的完整 DCF 假设集

**推荐的整合列模式（使用 INDEX）：**
`=INDEX(B10:D10, 1, $B$6)`

**不是这样 - 散布的 IF 语句：**
`=IF($B$6=1,[熊市块单元格],IF($B$6=2,[基准块单元格],[牛市块单元格]))`

整合列方法集中逻辑，使模型更易于审计。

### 正确的收入预测模式

**使用 INDEX 公式创建整合列，然后在预测中引用它：**

**步骤 1 - 用于 FY1 增长的整合列：**
`=INDEX([熊市 FY1 增长]:[牛市 FY1 增长], 1, $B$6)`

**步骤 2 - 收入预测引用整合列：**
`收入 第 1 年： =D29*(1+$E$10)`

其中：
- D29 = 上一年收入
- $E$10 = 用于 FY1 增长的整合列单元格（包含 INDEX 公式）
- $B$6 = 情景选择器（1=熊市，2=基准，3=牛市）

**这种方法比在每个预测公式中嵌入 IF 语句更清晰**，并且更容易审计使用了哪个情景假设。

### 正确的自由现金流公式模式

**使用 INDEX 公式的整合列，然后在自由现金流计算中引用它们：**

**整合列方法：**
```csv
项目，公式，引用
折旧摊销，=E29*$E$21，$E$21 = 用于折旧摊销百分比的整合列
资本支出，=E29*$E$22，$E$22 = 用于资本支出百分比的整合列
净营运资本变动，=(E29-D29)*$E$23，$E$23 = 用于净营运资本百分比的整合列
无杠杆自由现金流，=E57+E58-E60-E62，E57=税后净营业利润 E58=折旧摊销 E60=资本支出 E62=净营运资本变动
```

**每个整合列单元格包含一个 INDEX 公式**，根据情景选择器从相应的情景块中提取数据。这使预测公式保持清晰且可审计。

在编写公式之前，确认情景块的行位置并设置好整合列。

### 正确的单元格批注格式

**每个硬编码值都需要这种格式：**

“来源：[系统/文档]，[日期]，[引用]，[如果适用，提供网址]”

**示例：**
```csv
项目，来源批注
股价，来源：市场数据脚本 2025-10-12 收盘价
流通股数，来源：10-K FY2024 第 45 页 注释 12
历史收入，来源：10-K FY2024 第 32 页 合并报表
贝塔值，来源：市场数据脚本 2025-10-12 5 年期月度贝塔值
一致预期，来源：管理层指引 2024 年第三季度财报电话会议
```

### 正确的假设表结构

**关键：每个情景块需要三个结构元素：**

1.  **部分标题行**（合并单元格）：例如，“熊市情景假设”
2.  **列标题行**显示年份 - **这是必需的，不要跳过**
3.  **数据行**包含假设值

**结构：**
```csv
熊市情景假设（部分标题 - 跨列 A:G 合并）
假设，FY1，FY2，FY3，FY4，FY5
收入增长率（%），X%，X%，X%，X%，X%
营业利润率（%），X%，X%，X%，X%，X%
永续增长率，X%，，，，
WACC，X%，，，，

基准情景假设（部分标题 - 跨列 A:G 合并）
假设，FY1，FY2，FY3，FY4，FY5
收入增长率（%），X%，X%，X%，X%，X%
营业利润率（%），X%，X%，X%，X%，X%
永续增长率，X%，，，，
WACC，X%，，，，

牛市情景假设（部分标题 - 跨列 A:G 合并）
假设，FY1，FY2，FY3，FY4，FY5
收入增长率（%），X%，X%，X%，X%，X%
营业利润率（%），X%，X%，X%，X%，X%
永续增长率，X%，，，，
WACC，X%，，，，
```

**如果没有显示预测年份（FY2025E, FY2026E 等）的列标题行，用户无法判断哪个假设值对应哪个年份。此行是强制性的。**

**然后创建一个整合列**（通常在右侧下一列），使用 INDEX 公式根据情景选择器从选定的情景块中提取数据。这个整合列就是你的预测公式所引用的。

### 正确的行规划流程

**1. 首先写入所有标题和标签：**
```csv
行，内容

2，股票代码 | 日期 | 财年结束日
4，情景选择器
7，关键假设
26，假设标题

```

**2. 写入所有分隔符和空白行**

**3. 然后使用锁定的行位置编写公式**

**4. 在创建后立即测试公式**

**把它想象成建造：**
- 好的：先浇筑地基，然后砌墙（结构稳固）
- 坏的：先砌墙，然后浇筑地基（墙会倒塌）

**Excel 版本：**
- 好的：先添加标题，然后编写公式（公式稳定）
- 坏的：先编写公式，然后添加标题（公式会出错）

### 正确的敏感性表实现

**重要提示**：这些**不是** Excel 的“数据表”功能。这些是简单的网格，您需要使用 openpyxl 编写常规公式。是的，这意味着总共大约 75 个公式（3 个表 × 每个 25 个单元格），但这很简单且是必需的。

**使用公式进行编程填充：**

每个敏感性表必须用公式完全填充，这些公式为每个假设组合重新计算隐含的股价。**不要使用 Excel 的数据表功能**（它需要手动干预，无法通过 openpyxl 自动化）。

**实现方法 - 具体示例：**

**表格结构 — 5×5 网格（奇数维度，基准情景居中）：**

如果模型的基准 WACC = 9.0%，基准永续增长率 = 3.0%，则围绕这些值对称地构建轴：

```csv
WACC vs 永续增长率，  2.0%，  2.5%，  3.0%，  3.5%，  4.0%
              8.0%，       [公式], [公式], [公式], [公式], [公式]
              8.5%，       [公式], [公式], [公式], [公式], [公式]
              9.0%，       [公式], [公式], [★  ], [公式], [公式]   ← 中间行 = 基准 WACC
              9.5%，       [公式], [公式], [公式], [公式], [公式]
             10.0%，       [公式], [公式], [公式], [公式], [公式]
                                   ↑
                          中间列 = 基准永续增长率 g
```

**★ = 中心单元格。** 其公式输出**必须**等于模型的实际隐含股价（来自估值摘要）。将此单元格应用中等蓝色填充（`#BDD7EE`）和粗体字，以便基准情景在视觉上锚定。

**轴值规则：** `axis_values = [base - 2*step, base - step, base, base + step, base + 2*step]` —— 围绕基准对称，奇数保证有中心。

**公式模式 - 单元格 B88（WACC=8.0%，永续增长率=2.0%）：**

B88 中的公式应使用以下值重新计算隐含价格：
- 来自行标题的 WACC：`$A88` (8.0%)
- 来自列标题的永续增长率：`B$87` (2.0%)

**推荐方法：** 引用主要 DCF 计算，但替换这些值。

**公式结构示例：**
`=([使用 $A88 作为折现率的预测期自由现金流现值之和] + [使用 B$87 作为增长率和 $A88 作为 WACC 计算的终值现值] - [净债务]) / [流通股数]`

**关键 - 为 5x5 网格中的每个单元格编写公式（每个表 25 个单元格，总共 75 个单元格）。** 使用 openpyxl 在循环中以编程方式编写这些公式。**不要**跳过此步骤或留下占位符文本。

**Python 实现模式：**
```python
# 用于填充敏感性表的伪代码
for row_idx, wacc_value in enumerate(wacc_range):
    for col_idx, term_growth_value in enumerate(term_growth_range):
        # 构建使用 wacc_value 和 term_growth_value 的公式
        formula = f"=<使用 {wacc_value} 和 {term_growth_value} 的 DCF 重新计算>"
        ws.cell(row=start_row+row_idx, column=start_col+col_idx).value = formula
```

**敏感性表在模型打开时必须立即工作，无需用户执行任何手动步骤。**

&lt;common_mistakes>

本节包含构建 DCF 模型时要避免的**所有错误模式**。

### 错误：简化或近似的敏感性表或占位符文本

**不要使用线性近似：**

```
// 错误 - 线性近似
B97: =B88*(1+(0.096-0.116))    // 假设线性关系

// 错误 - 除法捷径
B105: =B88/(1+(E48-0.07))      // 没有重新计算完整的 DCF
```

**不要留下占位符文本：**
```
// 错误 - 占位符注释
“注意：使用 Excel 数据表功能（数据 → 假设分析 → 数据表）来填充敏感性表。”

// 错误 - 空单元格
[因“这很复杂”而留空单元格]
```

**不要混淆术语：**
- ❌ “敏感性表需要 Excel 的数据表功能”（不 - 这是一个我们无法使用的特定 Excel 工具）
- ✅ “敏感性表是包含公式的简单网格”（是的 - 这是我们构建的内容）

**为什么这些捷径是错误的：**
- 线性近似公式实际上没有重新计算 DCF - 它们只是应用简单的数学调整
- 这些关系不是线性的，因此结果将不准确
- 占位符文本需要用户手动干预
- 交付时模型不能立即使用
- 不专业或不符合客户要求
- 空单元格 = 未完成的交付物

**常见的要拒绝的合理化说法：**
“编写 75 多个公式感觉很复杂，所以我留个注释让用户手动完成。”

**现实情况：** 当您在 Python 中使用 openpyxl 编写循环时，编写 75 个公式是直接了当的。每个公式遵循相同的模式 - 只需替换行/列值。这是交付物中必需的部分。

**相反：** 用公式填充每个敏感性单元格，为该特定假设组合重新计算完整的 DCF

### 错误：缺少单元格批注

**不要这样做：**
- 创建所有硬编码输入时不加批注
- 想着“我稍后再加”
- 写上“待办：添加来源”
- 留下蓝色输入单元格没有文档记录

**为什么是错误的：**
- 无法验证数据来源
- 不符合 xlsx 技能要求
- 未经审计就绪
- 之后修复浪费时间

**相反：** 在创建每个硬编码值**时**就添加单元格批注

### 错误：公式行引用错误

**症状：**
自由现金流部分引用了错误的假设行：
`折旧摊销：  =E29*$E$34    // 应该是 $E$21，但引用了错误的行`
`资本支出： =E29*$E$41   // 应该是 $E$22，但行发生了偏移`

**发生这种情况的原因：**
1.  先编写公式
2.  然后插入标题
3.  所有行引用发生了偏移
4.  现在公式指向了错误的单元格 → #REF! 错误

**相反：** 首先锁定行布局，然后编写公式

### 错误：每个假设在所有情景中使用单行

**不要这样构建假设：**
```csv
假设，熊市，基准，牛市
收入增长 FY1，10%，13%，16%
收入增长 FY2，9%，12%，15%
```
这种垂直布局使得难以查看每个情景中跨年份的进展。

**为什么是错误的：**
- 使得难以查看每个情景中假设跨年份的演变
- 更难比较整个预测期内的情景假设
- 对于审查情景逻辑来说不太直观

**相反：**
- 为每个情景（熊市、基准、牛市）创建单独的块
- 在每个块中，跨预测年份水平显示假设
- 这使得每个情景的假设作为一个整体更容易审查

### 错误：没有边框

**不要交付没有边框的模型：**
- 没有部分划分
- 所有单元格混在一起
- 难以阅读且不专业

**为什么是错误的：**
- 不符合客户要求
- 难以导航
- 看起来业余

**相反：** 为所有主要部分添加边框

### 错误：字体颜色错误或没有字体颜色区分

**不要这样做：**
- 所有文本都是黑色
- 只使用填充颜色（没有字体颜色变化）
- 搞混哪些单元格是蓝色还是黑色

**为什么是错误的：**
- 无法区分输入和公式
- 审计变得不可能
- 违反 xlsx 技能要求

**相反：** 所有硬编码输入用蓝色文本，所有公式用黑色文本，工作表链接用绿色

### 错误：运营费用基于毛利

**不要这样做：**
`销售与营销： =E33*0.15    // E33 = 毛利（错误）`

**为什么是错误的：**
- 运营费用随收入规模扩大，而不是毛利
- 产生不切实际的利润走势
- 企业实际运营方式并非如此

**相反：**
`销售与营销： =E29*0.15    // E29 = 收入（正确）`

### 前 5 个错误总结

1.  **公式行引用错误** → 在编写公式**之前**定义**所有**行位置
2.  **缺少单元格批注** → 在单元格创建**时**添加批注，而不是最后
3.  **简化的敏感性表** → 用完整的 DCF 重新计算公式填充所有单元格，而不是近似值
4.  **情景块引用错误** → 确保 IF 公式从正确的熊市/基准/牛市块中提取数据
5.  **没有边框** → 添加专业的部分边框以达到客户要求的外观

此外，请注意这些错误：

### WACC 计算错误
- 在资本结构中混合账面价值和市场价值
- 错误地使用股权贝塔而非资产/无杠杆贝塔
- 对债务成本应用了错误的税率
- 无风险利率错误（必须使用当前 10 年期国债）
- 未能根据净债务 vs 净现金头寸进行调整

### 增长假设缺陷
- 永续增长率 > WACC（导致价值为无穷大）
- 预测增长率与历史表现不一致
- 忽视行业增长限制
- 收入增长与单位经济效益不匹配
- 没有运营依据的利润扩张

### 终值错误
- 使用错误的增长方法（永续增长 vs 退出乘数）
- 终值 > 企业价值的 80%（表明过度依赖）
- 与稳态假设不一致的终期利润
- 终值的折现期间错误

### 现金流预测错误
- 运营费用基于毛利而非收入
- 折旧摊销/资本支出百分比与商业模式不一致
- 营运资本变化计算不当
- 年份间税率不一致
- 税后净营业利润计算错误

**这些是最常见的错误。在开始任何 DCF 构建之前，请重新阅读本节。**

每个 DCF 模型必须最大化满足以下要求：
1. **基于历史表现的真实收入和利润率假设**
2. **使用标准 CAPM 方法计算适当的资本成本**

# DCF 模型构建指南

## 所需输入要求

### 最低必需输入
1. **公司标识**：股票代码或公司名称
2. **增长假设**：预测期的营收增长率（或“使用共识预期”）
3. **可选参数**：
   - 预测期（默认：5年）
   - 情景案例（熊市/基准/牛市下的增长及利润率假设）
   - 终值增长率（默认：2.5-3.0%）
   - 若不使用CAPM，需提供具体的WACC输入参数

## Excel 模型结构

### 工作表架构

创建**两个工作表**：

1. **DCF** - 包含敏感性分析的主估值模型（位于底部）
2. **WACC** - 资本成本计算表

**关键要求**：敏感性分析表放在 **DCF 工作表的底部**（而非单独的工作表）。这能确保所有估值输出集中在一起。

### 公式重算（强制要求）

创建或修改Excel模型后，**必须使用 `excel-author` 技能中的 `recalc.py` 脚本重算所有公式**：

```bash
python recalc.py [Excel文件路径] [超时秒数]
```

示例：
```bash
python recalc.py AAPL_DCF_Model_2025-10-12.xlsx 30
```

该脚本将：
- 使用LibreOffice重算所有工作表中的公式
- 扫描所有单元格查找Excel错误（#REF!, #DIV/0!, #VALUE!, #NAME?, #NULL!, #NUM!, #N/A）
- 返回包含错误位置和数量的详细JSON

**预期输出格式：**
```json
{
  "status": "success",           // 或 "errors_found"
  "total_errors": 0,              // 总错误数
  "total_formulas": 42,           // 文件中的公式总数
  "error_summary": {}             // 仅当发现错误时出现
}
```

**如发现错误**，输出将包含详情：
```json
{
  "status": "errors_found",
  "total_errors": 2,
  "total_formulas": 42,
  "error_summary": {
    "#REF!": {
      "count": 2,
      "locations": ["DCF!B25", "DCF!C25"]
    }
  }
}
```

**修复所有错误**，并重新运行 `recalc.py`，直至状态为 "success" 后再交付模型。

### 格式化标准

**重要提示**：遵循 xlsx 技能中关于公式构建规则和数字格式惯例。DCF 技能增加了具体的视觉呈现标准。

**配色方案 - 双层结构**：

**第1层：字体颜色（xlsx技能强制要求）**
- **蓝色文字（RGB: 0,0,255）**：所有硬编码输入（股价、股本、历史数据、假设）
- **黑色文字（RGB: 0,0,0）**：所有公式和计算结果
- **绿色文字（RGB: 0,128,0）**：链接到其他工作表的引用（WACC工作表引用）

**第2层：填充颜色 - 专业蓝/灰调色板（除非用户另有指定，否则使用默认）**
- **保持简洁** — 仅使用蓝色和灰色进行填充。切勿引入绿色、黄色、橙色或多种强调色。过多颜色会让模型显得不专业。
- **默认填充色板：**
  - **章节标题**：深蓝色背景（RGB: 31,78,121 / `#1F4E79`）配白色加粗文字
  - **子标题/列标题**：浅蓝色背景（RGB: 217,225,242 / `#D9E1F2`）配黑色加粗文字
  - **输入单元格**：浅灰色背景（RGB: 242,242,242 / `#F2F2F2`）配蓝色字体 — 或为追求极简，使用白色背景配蓝色字体
  - **计算单元格**：白色背景配黑色字体
  - **输出/汇总行**（每股价值、企业价值等）：中蓝色背景（RGB: 189,215,238 / `#BDD7EE`）配黑色加粗字体
- **仅此而已 — 3种蓝色 + 1种灰色 + 白色。** 克制添加更多颜色的冲动。
- 用户提供的模板或明确的颜色偏好始终覆盖这些默认设置。

**两层如何协同工作：**
- 输入单元格：蓝色字体 + 浅灰色填充 = “硬编码输入”
- 公式单元格：黑色字体 + 白色背景 = “计算值”
- 工作表链接：绿色字体 + 白色背景 = “来自其他工作表的引用”
- 关键输出：黑色加粗字体 + 中蓝色填充 = “这是答案”

**字体颜色告诉你它是什么（输入/公式/链接）。填充颜色告诉你它在哪里（标题/数据/输出）。**

**关键WACC公式：**
```
市值 = 股价 × 股本
净债务 = 总债务 - 现金
企业价值 = 市值 + 净债务
股权权重 = 市值 / 企业价值
债务权重 = 净债务 / 企业价值
WACC = (股权成本 × 股权权重) + (税后债务成本 × 债务权重)
```

### 敏感性分析（位于DCF工作表底部）

**术语提醒**：“敏感性表” = 简单的二维网格，包含行标题、列标题和每个数据单元格中的公式。**非** Excel的“数据表”功能（数据 → 模拟分析 → 数据表）。您将使用openpyxl将常规Excel公式写入每个单元格。

**位置**：DCF工作表第87行起（**非**单独工作表）

**三个垂直堆叠的敏感性表：**

1. **WACC vs 终值增长率**（第87-100行）- 5x5网格 = 25个带公式的单元格
2. **营收增长率 vs EBIT利润率**（第102-115行）- 5x5网格 = 25个带公式的单元格
3. **贝塔系数 vs 无风险利率**（第117-130行）- 5x5网格 = 25个带公式的单元格

**公式总数：75个**（此为必需要求，非可选）

**关键要求**：所有敏感性表单元格必须使用openpyxl以编程方式填充公式。**不要**使用线性近似捷径。**不要**留下占位符文本或关于手动步骤的说明。**不要**以“这很复杂”为由让单元格留空 — 使用Python循环生成公式。

**表格设置：**
1. 创建包含行/列标题的表结构（要测试的假设值）
2. 用公式填充**每一个**数据单元格，该公式应：
   - 使用行标题值（例如，WACC = 9.0%）

6. 表格之间留1-2个空行

**无需手动干预** — 用户打开文件时，敏感性表必须完全可用。

## 情景选择器实现

**三种情景框架：**

### 炒作情景
- 保守的收入增长（历史范围低端）
- 利润率压缩或未扩张
- 较高的 WACC（风险溢价增加）
- 较低的终端增长率
- 较高的资本支出假设

### 基准情景
- 符合共识或管理层指引的收入增长
- 基于运营杠杆的适度利润率扩张
- 当前市场隐含的 WACC
- 与 GDP 一致的终端增长率（2.5-3.0%）
- 标准资本支出假设

### 乐观情景
- 乐观的收入增长（预测范围高端）
- 显著的利润率扩张
- 较低的 WACC（风险溢价降低）
- 较高的终端增长率（3.5-5.0%）
- 较低的资本支出强度

**公式实现：**

**不要使用分散嵌套的 IF 公式。** 相反，应创建一个汇总列，使用 INDEX 或 OFFSET 公式从相应的情景模块中提取数据。

**推荐模式（使用 INDEX）：**
`=INDEX(B10:D10, 1, $B$6)`，其中 `B10:D10` = 炒作/基准/乐观情景值，`1` = 行偏移量，`$B$6` = 情景选择器单元格（1、2 或 3）

**然后在所有预测中引用该汇总列：**
`收入第 1 年： =D29*(1+$E$10)`，其中 $E$10 是第 1 年增长率的汇总列值。

这种方法集中了情景逻辑，使模型更易于审计和维护。

## 常见变体

### 高增长科技公司
- 较长的预测期（7-10 年）
- 较高的初始增长率（20-30%）
- 随时间推移的显著利润率扩张
- 较高的 WACC（12-15%）
- 对单位经济学进行建模（用户、ARPU 等）

### 成熟/稳定公司
- 较短的预测期（3-5 年）
- 适中的增长率（GDP +1-3%）
- 稳定的利润率
- 较低的 WACC（7-9%）
- 关注现金生成和资本配置

### 周期性公司
- 贯穿经济周期进行建模
- 在周期中段正常化利润率
- 考虑低谷和峰值情景
- 根据周期性调整贝塔值

### 多元业务公司
- 为每个业务单元单独进行 DCF
- 不同业务分部采用不同的增长率和利润率
- 分部加总估值
- 考虑协同效应

- **用户提供的数据**：历史财务数据、共识预期
- **手动提取**：SEC EDGAR 文件作为后备方案

## 最终输出检查清单

交付 DCF 模型前：

**必需项：**
- 运行 `python recalc.py model.xlsx 30` 直到状态为 "success"（零公式错误）
- 两个工作表：DCF（底部包含敏感性分析）、WACC
- 字体颜色：蓝色=输入，黑色=公式，绿色=工作表链接
- 所有硬编码输入均有单元格注释
- 敏感性分析表完全由公式填充
- 主要部分周围有专业边框

**验证项：**
- 营业费用基于收入（而非毛利润）
- 终端价值占企业价值的 50-70%
- 终端增长率 < WACC
- 税率 21-28%
- 文件命名：`[股票代码]_DCF_模型_[日期].xlsx`
