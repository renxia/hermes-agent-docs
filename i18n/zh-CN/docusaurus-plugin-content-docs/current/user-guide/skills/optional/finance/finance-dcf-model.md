# DCF 模型构建器

## 概述
本技能旨在创建符合投资银行标准的股权估值机构级 DCF 模型。每次分析都会生成一个详细的 Excel 模型（敏感性分析包含在 DCF 工作表底部）。

## 工具
- 默认使用用户提供的所有信息以及可用于数据获取的 MCP 服务器。

## 关键约束 - 请先阅读这些内容
以下约束适用于所有 DCF 模型构建。开始前请仔细审阅：

**公式优于硬编码（不可协商）：**
- 每个预测值、利润率、贴现因子、现值和敏感性单元格都必须是实时 Excel 公式——绝不能是 Python 计算后写入的数值
- 使用 openpyxl 时：`ws["D20"] = "=D19*(1+$B$8)"` 是正确的；`ws["D20"] = calculated_revenue` 是错误的
- 唯一允许的硬编码数字是：(1) 原始历史输入，(2) 假设驱动因素（增长率、WACC 输入值、永续增长率 g），(3) 当前市场数据（股价、债务余额）
- 如果你发现自己在 Python 中计算某个值然后写入结果——请停止。模型必须在用户更改假设时能够灵活调整。

**与用户逐步验证（不要端到端构建）：**
- 数据检索后 → 向用户展示原始输入数据块（收入、利润率、股份数、净债务）并在建模前确认
- 收入预测后 → 展示预测的营收总额和增长率，在构建利润率模型前确认
- 自由现金流模型完成后 → 展示完整的自由现金流时间表，在计算 WACC 前确认逻辑
- WACC 计算后 → 展示计算过程和输入值，在贴现前确认
- 终值 + 现值计算后 → 展示权益桥接（企业价值 → 权益价值 → 每股价值），在制作敏感性表前确认
- 在每个阶段捕获错误——如果在敏感性表构建完成后才发现错误的利润率假设，意味着需要重建所有下游内容

**敏感性表：**
- **使用奇数行数和列数**（标准：5×5，有时 7×7）——这保证了有一个真正的中心单元格
- **中心单元格 = 基本情景。** 构建轴值时，确保中间行标题和中间列标题与模型的实际假设完全相等（例如，如果基础 WACC = 9.0%，则中间行为 9.0%；如果永续增长率 g = 3.0%，则中间列为 3.0%）。因此，中心单元格的输出必须等于模型实际隐含的股价——这是验证表格构建正确性的关键检查点。
- **高亮中心单元格**：使用中蓝色填充（`#BDD7EE`）加粗字体，以便立即可见哪个单元格是基本情景。
- 为所有单元格（通常为 3 个表 × 25 个单元格 = 75 个）填充完整的 DCF 重计算公式
- 使用 openpyxl 循环以编程方式编写公式
- 无占位符文本，无线性近似，无需手动操作
- 每个单元格必须针对该假设组合重新计算完整的 DCF

**单元格注释：**
- 在创建每个硬编码值时立即添加单元格注释
- 格式："来源：[系统/文档], [日期], [参考], [URL（如适用）]"
- 每个蓝色输入值在进入下一节前都必须有注释
- 不要推迟到最后或写入 "TODO: 添加来源"

**模型布局规划：**
- 在编写任何公式之前，定义所有节的行位置
- 首先编写所有标题和标签
- 其次编写所有分节符和空行
- 然后使用锁定的行位置编写公式
- 创建公式后立即进行测试

**公式重计算：**
- 交付前运行 `python recalc.py model.xlsx 30`
- 修复所有错误，直到状态为 "success"
- 要求零公式错误（#REF!, #DIV/0!, #VALUE! 等）

**情景模块：**
- 为熊市/基准/牛市情景创建独立模块
- 在每个模块内水平显示各预测年度的假设值
- 使用 IF 公式：`=IF($B$6=1,[熊市单元格],IF($B$6=2,[基准单元格],[牛市单元格]))`
- 验证公式引用了正确情景模块的单元格

## DCF 流程工作流

### 第 1 步：数据检索与验证
从 MCP 服务器、用户提供的数据和网络获取数据。

**数据来源优先级：**
1. **MCP 服务器**（如果已配置） - 来自 Daloopa 等提供商的结构化财务数据
2. **用户提供的数据** - 来自他们研究的财务历史数据
3. **网络搜索/获取** - 当前股价、贝塔系数、债务和现金（需要时）

**验证清单：**
- 验证净债务与净现金（对估值至关重要）
- 确认摊薄流通股数（检查近期回购/发行情况）
- 验证历史利润率是否符合商业模式
- 将收入增长率与行业基准进行交叉核对
- 验证税率是否合理（通常为 21-28%）

### 第 2 步：历史分析（3-5 年）
分析并记录：
- **收入增长趋势**：计算 CAGR，识别驱动因素
- **利润率趋势**：跟踪毛利率、EBIT 利润率、FCF 利润率
- **资本密集度**：折旧摊销和资本支出占收入的百分比
- **营运资金效率**：净营运资金变化占收入增长的百分比
- **回报指标**：ROIC、ROE 趋势

创建摘要表格显示：
```
历史指标（最近十二个月）：
收入：X 百万美元
收入增长：X% CAGR
毛利率：X%
EBIT 利润率：X%
折旧摊销占收入百分比：X%
资本支出占收入百分比：X%
FCF 利润率：X%
```

### 第 3 步：构建收入预测
**方法论：**
1. 从最近的实际收入（最近十二个月或最近财年）开始
2. 为每个预测年度应用增长率
3. 同时显示美元金额和计算出的增长百分比

**增长率框架：**
- 第 1-2 年：较高的增长，反映近期可见性
- 第 3-4 年：逐步向行业平均水平缓和
- 第 5 年+：接近永续增长率

**公式结构：**
- 收入(第 N 年) = 收入(第 N-1 年) × (1 + 增长率)
- 增长率%(第 N 年) = 收入(第 N 年) / 收入(第 N-1 年) - 1

**三种情景方法：**
```
熊市情景：保守增长（例如，8-12%）
基准情景：最可能情景（例如，12-16%）
牛市情景：乐观增长（例如，16-20%）
```

### 第 4 步：运营费用建模
**固定/可变成本分析：**

运营费用应模拟实际的运营杠杆：
- **销售与市场营销**：通常占收入的 15-40%，具体取决于商业模式
- **研发**：科技公司通常为 10-30%
- **一般及行政费用**：通常占收入的 8-15%，随公司规模扩大显示出杠杆效应

**关键原则：**
- 所有百分比基于收入，而非毛利润
- 模拟运营杠杆：百分比应随收入增长而下降
- 为销售与市场营销、研发、一般及行政费用维护独立的行项目
- 计算 EBIT = 毛利润 - 总运营费用

**利润率扩张框架：**
```
当前状态 → 目标状态（第 5 年）
毛利率：X% → Y%（基于规模、效率证明合理性）
EBIT 利润率：X% → Y%（收入增长 + 运营费用杠杆的结果）
```

### 第 5 步：自由现金流计算
**按适当顺序构建自由现金流：**

```
EBIT
(-) 税项（EBIT × 税率）
= 税后净营业利润
(+) 折旧摊销（非现金费用，占收入的百分比）
(-) 资本支出（占收入的百分比，通常为 4-8%）
(-) 净营运资金变化（营运资金变动）
= 无杠杆自由现金流
```

**营运资金建模：**
- 计算为收入变化（收入增量）的百分比
- 典型范围：收入变化的 -2% 至 +2%
- 负数 = 现金来源（营运资金释放）
- 正数 = 现金使用（营运资金积累）

**维护性 vs 增长性资本支出：**
- 维护性资本支出：维持当前运营（约占收入的 2-3%）
- 增长性资本支出：支持扩张（额外占收入的 2-5%）
- 总资本支出应与公司的增长战略保持一致

### 第 6 步：资本成本（WACC）研究
**权益成本的 CAPM 方法论：**

```
权益成本 = 无风险利率 + 贝塔系数 × 权益风险溢价

其中：
- 无风险利率 = 当前 10 年期国债收益率
- 贝塔系数 = 5 年月度股票贝塔系数 vs 市场指数
- 权益风险溢价 = 5.0-6.0%（市场标准）
```

**债务成本计算：**

```
税后债务成本 = 税前债务成本 × (1 - 税率)

根据以下信息确定税前债务成本：
- 信用评级（如果可用）
- 公司债券当前收益率
- 财务数据中的利息费用 / 总债务
```

**资本结构权重：**

```
权益市场价值 = 当前股价 × 流通股数
净债务 = 总债务 - 现金及等价物
企业价值 = 市值 + 净债务

权益权重 = 市值 / 企业价值
债务权重 = 净债务 / 企业价值

WACC = (权益成本 × 权益权重) + (税后债务成本 × 债务权重)
```

**特殊情况：**
- **净现金头寸**：如果现金 > 债务，净债务为负数
  - 债务权重可能为负数
  - WACC 计算相应调整
- **无债务**：WACC = 权益成本

**典型 WACC 范围：**
- 大盘股，稳定型：7-9%
- 成长型公司：9-12%
- 高增长/高风险型：12-15%

### 第 7 步：贴现率应用（5-10 年预测）
**年中惯例：**
- 假定现金流在年中发生
- 贴现期：0.5, 1.5, 2.5, 3.5, 4.5, 等等
- 贴现因子 = 1 / (1 + WACC)^期数

**现值计算：**
```
对于每个预测年度：
FCF 的现值 = 无杠杆 FCF × 贴现因子

示例（第 1 年）：
FCF = 1,000 美元
WACC = 10%
期数 = 0.5
贴现因子 = 1 / (1.10)^0.5 = 0.9535
现值 = 1,000 美元 × 0.9535 = 954 美元
```

**预测期选择：**
- **5 年**：大多数分析的标准
- **7-10 年**：具有较长增长跑道的高增长公司
- **3 年**：成熟、稳定的业务

### 第 8 步：终值计算
**永续增长法（首选）：**

```
终期 FCF = 最后一年 FCF × (1 + 永续增长率)
终值 = 终期 FCF / (WACC - 永续增长率)

关键约束：永续增长率 < WACC（否则价值为无穷大）
```

**永续增长率选择：**
- 保守：2.0-2.5%（GDP 增长率）
- 中等：2.5-3.5%
- 激进：3.5-5.0%（仅适用于市场领导者）

**不要超过**：无风险利率或长期 GDP 增长率

**退出倍数法（替代方法）：**
```
终值 = 最后一年 EBITDA × 退出倍数

其中退出倍数来源于：
- 可比公司交易倍数
- 先例交易倍数
- 典型范围：8-15 倍 EBITDA
```

**终值的现值：**
```
终值的现值 = 终值 / (1 + WACC)^最终期数

其中最终期数考虑时间因素：
5 年模型，使用年中惯例：期数 = 4.5
```

**终值合理性检查：**
- 应占企业价值的 50-70%
- 如果 >75%，模型可能过度依赖终值假设
- 如果 &lt;40%，检查终值假设是否过于保守

### 第 9 步：企业价值到权益价值的桥接
**估值摘要结构：**

```
(+) 预测 FCF 现值总和 = X 百万美元
(+) 终值现值 = Y 百万美元
= 企业价值 = Z 百万美元

(-) 净债务 [或 + 净现金（如果为负）] = A 百万美元
= 权益价值 = B 百万美元

÷ 摊薄流通股数 = C 百万股
= 隐含每股股价 = XX.XX 美元

当前股价 = YY.YY 美元
隐含回报 = (隐含价格 / 当前价格) - 1 = XX%
```

**关键调整：**
- **净债务 = 总债务 - 现金及等价物**
  - 如果为正：从企业价值中扣除（减少权益价值）
  - 如果为负（净现金）：加到企业价值中（增加权益价值）
- **使用摊薄股数**：包括期权、限制性股票单位、可转换证券
- **其他调整**（如适用）：
  - 少数股东权益
  - 养老金负债
  - 经营租赁负债

**估值输出格式：**
```csv
估值组成部分,金额（百万美元）
预测 FCF 现值,X.X
终值现值,Y.Y
企业价值,Z.Z
(-) 净债务,A.A
权益价值,B.B
,,
流通股数（百万股）,C.C
隐含每股股价,$XX.XX
当前股价,$YY.YY
隐含上涨/下跌空间,+XX%
```

### 第 10 步：敏感性分析
在 DCF 工作表底部构建**三个敏感性表**，显示估值如何随不同假设变化：

1. **WACC vs 永续增长率** - 显示企业价值对贴现率和永续增长的敏感性
2. **收入增长 vs EBIT 利润率** - 显示营收增长和运营杠杆的影响
3. **贝塔系数 vs 无风险利率** - 显示对权益成本组成部分的敏感性

**实现方法**：这些是简单的二维网格（不是 Excel 的 "数据表" 功能），每个单元格中都有公式。每个单元格必须包含针对该特定假设组合的完整 DCF 重计算公式。有关使用 openpyxl 以编程方式填充所有 75 个单元格的详细要求，请参阅关键约束部分。

### 情景模块选择模式 - 遵循此方法

**假设按每个情景的独立模块组织：**

**关键结构 - 每个节标题三行：**

```csv
熊市情景假设（节标题，跨列合并单元格）
假设,FY1,FY2,FY3,FY4,FY5
收入增长率（%）,12%,10%,9%,8%,7%
EBIT 利润率（%）,45%,44%,43%,42%,41%

基准情景假设（节标题，跨列合并单元格）
假设,FY1,FY2,FY3,FY4,FY5
收入增长率（%）,16%,14%,12%,10%,9%
EBIT 利润率（%）,48%,49%,50%,51%,52%

牛市情景假设（节标题，跨列合并单元格）
假设,FY1,FY2,FY3,FY4,FY5
收入增长率（%）,20%,18%,15%,13%,11%
EBIT 利润率（%）,50%,51%,52%,53%,54%
```

**每个情景模块必须有一个列标题行**，显示预测年度（FY2025E, FY2026E 等），紧接在节标题下方。没有此行，用户无法判断哪个假设值对应哪个年度。

**如何引用假设 - 创建合并列：**
1. 情景选择器单元格（例如 B6）包含 1=熊市, 2=基准, 或 3=牛市
2. 创建一个包含 INDEX 或 OFFSET 公式的合并列，以从正确的情景模块中提取数据
3. 预测公式引用该合并列（清晰的单元格引用）
4. 每个情景模块包含所有预测年度的完整 DCF 假设集

**推荐的合并列模式（使用 INDEX）：**
`=INDEX(B10:D10, 1, $B$6)`

**不要这样做 - 到处使用分散的 IF 语句：**
`=IF($B$6=1,[熊市模块单元格],IF($B$6=2,[基准模块单元格],[牛市模块单元格]))`

合并列方法集中了逻辑，使模型更易于审计。

### 正确的收入预测模式

**创建包含 INDEX 公式的合并列，然后在预测中引用它：**

**第 1 步 - FY1 增长的合并列：**
`=INDEX([熊市 FY1 增长]:[牛市 FY1 增长], 1, $B$6)`

**第 2 步 - 收入预测引用合并列：**
`第 1 年收入: =D29*(1+$E$10)`

其中：
- D29 = 上一年收入
- $E$10 = FY1 增长的合并列单元格（包含 INDEX 公式）
- $B$6 = 情景选择器（1=熊市, 2=基准, 3=牛市）

**此方法比在每个预测公式中嵌入 IF 语句更简洁**，并且更容易审计使用了哪些情景假设。

### 正确的 FCF 公式模式

**使用包含 INDEX 公式的合并列，然后在 FCF 计算中引用它们：**

**合并列方法：**

### 正确的敏感性表实现

**重要提示**：这些不是 Excel 的 "数据表" 功能。这些是简单的网格，您使用 openpyxl 编写常规公式。是的，这意味着总共大约 75 个公式（3 个表 × 每表 25 个单元格），但这很直接且是必需的。

**使用公式进行程序化填充：**

每个敏感性表必须用公式完全填充，这些公式重新计算每个假设组合的隐含股价。**不要使用 Excel 的数据表功能**（它需要手动干预，无法通过 openpyxl 自动化）。

**实现方法 - 具体示例：**

**表结构 — 5×5 网格（奇数维度，基本情景居中）：**

如果模型的基础 WACC = 9.0%，基础永续增长率 = 3.0%，则围绕这些值对称构建轴：

```csv
WACC vs 永续增长率,  2.0%,  2.5%,  3.0%,  3.5%,  4.0%
              8.0%,       [公式], [公式], [公式], [公式], [公式]
              8.5%,       [公式], [公式], [公式], [公式], [公式]
              9.0%,       [公式], [公式], [★  ], [公式], [公式]   ← 中间行 = 基础 WACC
              9.5%,       [公式], [公式], [公式], [公式], [公式]
             10.0%,       [公式], [公式], [公式], [公式], [公式]
                                   ↑
                          中间列 = 基础永续增长率 g
```

**★ = 中心单元格。** 其公式输出必须等于模型实际隐含的股价（来自估值摘要）。对此单元格应用中蓝色填充（`#BDD7EE`）和粗体字体，以便基础情景在视觉上被锚定。

**轴值规则：`axis_values = [基础值 - 2*步长, 基础值 - 步长, 基础值, 基础值 + 步长, 基础值 + 2*步长]` — 围绕基础值对称，奇数数量保证存在中心点。

**公式模式 - 单元格 B88（WACC=8.0%，永续增长率=2.0%）：**

B88 中的公式应使用以下值重新计算隐含价格：
- 来自行标题的 WACC：`$A88` (8.0%)
- 来自列标题的永续增长率：`B$87` (2.0%)

**推荐方法：** 引用主要的 DCF 计算，但替换这些值。

**示例公式结构：**
`=([使用 $A88 作为贴现率的 FCF 现值总和] + [使用 B$87 作为增长率和 $A88 作为 WACC 的终值] - [净债务]) / [股份数]`

**关键 - 为 5x5 网格中的每个单元格（每个表 25 个单元格，总共 75 个单元格）编写公式。** 使用 openpyxl 以编程方式在循环中编写这些公式。不要跳过此步骤或留下占位符文本。

**Python 实现模式：**
```python
# 用于填充敏感性表的伪代码
for row_idx, wacc_value in enumerate(wacc_range):
    for col_idx, term_growth_value in enumerate(term_growth_range):
        # 构建使用 wacc_value 和 term_growth_value 的公式
        formula = f"=<使用 {wacc_value} 和 {term_growth_value} 重计算 DCF>"
        ws.cell(row=start_row+row_idx, column=start_col+col_idx).value = formula
```

**敏感性表必须在模型打开时立即工作，无需用户进行任何手动操作。**

本节包含构建 DCF 模型时应避免的所有错误模式。

### 错误：简化的敏感性表近似或占位符文本

**不要使用线性近似：**

```
// 错误 - 线性近似
B97: =B88*(1+(0.096-0.116))    // 假设线性关系

// 错误 - 除法捷径
B105: =B88/(1+(E48-0.07))      // 没有重新计算完整的 DCF
```

**不要留下占位符文本：**
```
// 错误 - 占位符注释
"注意：使用 Excel 数据表功能（数据 → 假设分析 → 数据表）来填充敏感性表。"

// 错误 - 空单元格
[因为“这很复杂”而将单元格留空]
```

**不要混淆术语：**
- ❌ "敏感性表需要 Excel 的数据表功能"（否 - 那是一个我们无法使用的特定 Excel 工具）
- ✅ "敏感性表是带有公式在每个单元格中的简单网格"（是 - 这是我们构建的内容）

**这些捷径错误的原因：**
- 线性近似公式实际上没有重新计算 DCF - 它们只是应用简单的数学调整
- 关系不是线性的，因此结果将不准确
- 占位符文本需要用户手动干预
- 交付时模型不能立即使用
- 不专业或不适合客户使用
- 空单元格 = 未完成的交付物

**常见的合理化理由需要拒绝：**
"编写 75 多个公式感觉很复杂，所以我留下一个注释让用户手动完成。"

**现实：** 当您使用 Python 和 openpyxl 中的循环时，编写 75 个公式很简单。每个公式遵循相同的模式 - 只需替换行/列值。这是交付物的必需部分。

**相反：** 为每个敏感性单元格填充公式，为该特定假设组合重新计算完整的 DCF

### 错误：缺少单元格注释

**不要这样做：**
- 创建所有硬编码输入而没有注释
- 认为“我稍后再添加”
- 写入 "TODO: 添加来源"
- 将蓝色输入留在没有文档记录的状态

**为什么错误：**
- 无法验证数据来源
- 不符合 xlsx 技能要求
- 未准备好审计
- 稍后修复浪费时间

**相反：** 在创建每个硬编码值时立即添加单元格注释

### 错误：公式行引用错误

**症状：**
FCF 部分引用了错误的假设行：
`折旧摊销:  =E29*$E$34    // 应该是 $E$21，但引用了错误的行`
`资本支出: =E29*$E$41   // 应该是 $E$22，但行偏移了`

**发生这种情况的原因：**
1. 首先编写公式
2. 然后插入标题
3. 所有行引用都发生了偏移
4. 现在公式指向了错误的单元格 → #REF! 错误

**相反：** 首先锁定行布局，然后编写公式

### 错误：每种假设跨情景使用单行

**不要这样构建假设：**
```csv
假设,熊市,基准,牛市
FY1 收入增长率,10%,13%,16%
FY2 收入增长率,9%,12%,15%
```
这种垂直布局使得难以查看每种情景内跨年度的增长率进展。

**为什么错误：**
- 使得难以查看每种情景内跨年度的假设演变
- 更难比较整个预测期内的情景假设
- 审查情景逻辑时不够直观

**相反：**
- 为每种情景（熊市、基准、牛市）创建独立的模块
- 在每个模块内，水平显示各预测年度的假设值
- 这使得每种情景的假设更容易作为一个连贯的集合进行审查

### 错误：无边框

**不要交付没有边框的模型：**
- 没有节的划分
- 所有单元格混在一起
- 难以阅读且不专业

**为什么错误：**
- 不适合客户使用
- 难以导航
- 看起来业余

**相反：** 在所有主要节周围添加边框

### 错误：字体颜色错误或没有字体颜色区分

**不要这样做：**
- 所有文本都是黑色
- 只使用填充颜色（没有字体颜色变化）
- 混淆哪些单元格应该是蓝色或黑色

**为什么错误：**
- 无法区分输入和公式
- 审计变得不可能
- 违反 xlsx 技能要求

**相反：** 蓝色文本用于所有硬编码输入，黑色文本用于所有公式，绿色用于工作表链接

### 错误：运营费用基于毛利润

**不要这样做：**
`销售与市场营销: =E33*0.15    // E33 = 毛利润（错误）`

**为什么错误：**
- 运营费用随收入扩大，而非毛利润
- 产生不现实的利润率进展
- 不符合企业实际运营方式

**相反：**
`销售与市场营销: =E29*0.15    // E29 = 收入（正确）`

### 前 5 大错误摘要

1. **公式行引用错误** → 在编写公式之前定义所有行位置
2. **缺少单元格注释** → 在创建单元格时立即添加注释，而非最后
3. **简化的敏感性表** → 用完整的 DCF 重计算公式填充所有单元格，而非近似值
4. **情景模块引用错误** → 确保 IF 公式从正确的熊市/基准/牛市模块中提取
5. **无边框** → 添加专业的节边框以获得适合客户的外观

此外，请注意以下错误：

### WACC 计算错误
- 混合资本结构中的账面价值和市场价值
- 错误地使用权益贝塔而非资产/无杠杆贝塔
- 对债务成本应用了错误的税率
- 无风险利率错误（必须使用当前 10 年期国债）
- 未能根据净债务 vs 净现金头寸进行调整

### 增长假设缺陷
- 永续增长率 > WACC（产生无穷价值）
- 预测增长率与历史表现不一致
- 忽视行业增长限制
- 收入增长与单位经济性不符
- 利润率扩张缺乏运营依据

### 终值错误
- 使用了错误的增长方法（永续增长 vs 退出倍数）
- 终值 > 企业价值的 80%（表明过度依赖）
- 终期利润率与稳态假设不一致
- 终值的贴现期错误

### 现金流预测错误
- 运营费用基于毛利润而非收入
- 折旧摊销/资本支出百分比与商业模式不匹配
- 营运资金变化计算不当
- 年度间税率不一致
- 税后净营业利润计算错误

**这些是最常见的错误。在开始任何 DCF 构建之前，请重新阅读本节。**

### 格式标准

**重要**：遵循xlsx技能的公式构建规则和数字格式惯例。DCF技能对视觉呈现有特定要求。

**颜色方案 - 两层结构**：

**第一层：字体颜色（强制遵循xlsx技能规定）**
- **蓝色文本（RGB：0,0,255）**：所有硬编码输入（股价、股数、历史数据、假设条件）
- **黑色文本（RGB：0,0,0）**：所有公式和计算结果
- **绿色文本（RGB：0,128,0）**：链接到其他工作表的引用（WACC工作表引用）

**第二层：填充颜色 - 专业蓝灰色调（除非用户另有指定，否则使用默认方案）**
- **保持简约** —— 仅使用蓝色和灰色填充。不要引入绿色、黄色、橙色或多种强调色。过多的颜色会让模型显得不专业。
- **默认填充色板：**
  - **章节标题**：深蓝色背景（RGB：31,78,121 / `#1F4E79`）配白色加粗文字
  - **副标题/列标题**：浅蓝色背景（RGB：217,225,242 / `#D9E1F2`）配黑色加粗文字
  - **输入单元格**：浅灰色背景（RGB：242,242,242 / `#F2F2F2`）配蓝色字体 —— 或如需极致简约，可使用白色背景配蓝色字体
  - **计算单元格**：白色背景配黑色字体
  - **输出/汇总行**（每股价值、EV等）：中等蓝色背景（RGB：189,215,238 / `#BDD7EE`）配黑色加粗字体
- **以上即：3种蓝色 + 1种灰色 + 白色。** 请克制添加更多颜色的冲动。
- 用户提供的模板或明确的颜色偏好始终优先于这些默认设置。

**两层如何协同工作：**
- 输入单元格：蓝色字体 + 浅灰色填充 = "硬编码输入"
- 公式单元格：黑色字体 + 白色背景 = "计算值"
- 工作表链接：绿色字体 + 白色背景 = "来自其他工作表的引用"

**关键WACC公式：**
```
市值 = 股价 × 流通股数
净债务 = 总债务 - 现金
企业价值 = 市值 + 净债务
股权权重 = 市值 / 企业价值
债务权重 = 净债务 / 企业价值
WACC = (股权成本 × 股权权重) + (税后债务成本 × 债务权重)
```

## 案例选择器实现

**三情景框架：**

### 悲观情景
- 保守的营收增长（历史范围低端）

下文许多段落提到“使用S&P Kensho MCP / Daloopa MCP / FactSet MCP”。这些是来自原始Cowork插件上下文的商业金融数据MCP。在Hermes中：

- **如果您配置了任何结构化金融数据MCP**（Hermes支持MCP — 参见 `native-mcp` 技能），优先使用它来获取时点可比公司、先例交易和文件。
- **否则**，退而使用：
  - `web_search` / `web_extract` 访问SEC EDGAR (`https://www.sec.gov/cgi-bin/browse-edgar`) 获取美国文件
  - 公司投资者关系页面获取新闻稿、财报演示文稿
  - `browser_navigate` 访问交互式数据门户
  - 用户提供的数据（当上下文中没有时明确询问）
- **切勿捏造**。如果无法获取某个倍数、先例或文件编号，将该单元格标记为 `[无来源]` 并向用户说明。
