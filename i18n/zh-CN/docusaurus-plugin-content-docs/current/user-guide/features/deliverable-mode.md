# 可交付模式

当 Hermes 智能体运行在消息网关（Slack、Discord、Telegram、WhatsApp、Signal 等）内时，它可以将生成的文件直接传送到聊天中——不是让用户复制路径，而是作为原生附件。

图表会显示为内联图片。PDF 报告会显示为可下载的文件。电子表格会以 `.xlsx` 格式上传。智能体无需编写 `MEDIA:` 标签或进行任何特殊处理——它只需生成文件并在响应中提及该文件的绝对路径。网关会从文本中提取该路径，将其从可见消息中移除，并原生上传文件。

## 工作原理

三个部分协同工作：

1.  **智能体拥有生成文件的工具。** 包括用于通过 matplotlib 生成图表的 `execute_code`，用于生成 PDF 的 `latex-pdf-report` 技能，用于生成演示文稿的 `powerpoint` 技能，用于生成图像的 `image_generate`，用于生成音频的 `text_to_speech` 等。
2.  **网关扫描智能体的响应以查找文件路径。** 任何以受支持扩展名结尾的绝对路径（`/tmp/...`）或相对于主目录的路径（`~/...`）都会被提取。代码块和内联代码中的路径会被忽略，因此代码示例不会被破坏。
3.  **网关根据文件类型进行分发。** 如果平台支持，图片会嵌入为内联；视频会嵌入为内联；音频会路由到语音/音频附件；其他所有内容都作为文件附件上传。

## 支持的文件扩展名

| 类别 | 扩展名 | 交付方式 |
|---|---|---|
| 图片 | `.png .jpg .jpeg .gif .webp .bmp .tiff .svg` | 内联嵌入 |
| 视频 | `.mp4 .mov .avi .mkv .webm` | 内联嵌入（在支持的地方） |
| 音频 | `.mp3 .wav .ogg .m4a .flac` | 语音 / 音频附件 |
| 文档 | `.pdf .docx .doc .odt .rtf .txt .md` | 文件上传 |
| 数据 | `.xlsx .xls .csv .tsv .json .xml .yaml .yml` | 文件上传 |
| 演示文稿 | `.pptx .ppt .odp` | 文件上传 |
| 压缩包 | `.zip .tar .gz .tgz .bz2 .7z` | 文件上传 |
| 网页 | `.html .htm` | 文件上传 |

`.py`、`.log` 和其他源文件扩展名被有意排除，以避免智能体自动发送任意源文件；如果您想向用户发送代码，请使用代码块。

## 鼓励智能体生成工件

默认情况下，智能体不会主动生成工件——它需要知道何时该这样做。有两种方式可以引导它：

**按会话：** 明确请求（“把比较结果以图表形式发给我”、“把数据以 CSV 格式返回”），或者编写您自己的自定义指令/个性设置，使其倾向于在即时通讯平台上生成工件类型的回复。

**项目级别：** 将偏好添加到智能体所工作的项目中的 `AGENTS.md` / `CLAUDE.md` / `.cursorrules` 文件，或者添加到 `~/.hermes/config.yaml` 文件下 `agent.custom_instructions` 中的全局自定义指令。

智能体需要使用的机制很简单：将文件渲染到绝对路径（例如 `/tmp/q3-revenue.png`），并在回复中以纯文本形式提及该路径。网关会处理其余部分。围栏代码块或反引号内的路径会被忽略，因此代码示例不会被破坏。

## 看板：工件随完成通知一起传送

如果您使用 Hermes 的看板多智能体工作流，工作者可以在其 `kanban_complete` 调用中附加可交付文件：

```python
kanban_complete(
    summary="已渲染 Q3 收入图表和报告",
    artifacts=[
        "/tmp/q3-revenue.png",
        "/tmp/q3-report.pdf",
    ],
)
```

当网关通知器向在 Slack/Telegram 等中订阅了该任务的任何人发送“任务完成”消息时，它也会将每个工件作为原生附件上传到该聊天中。用户可以在一个地方同时收到可交付物和摘要。

如果在通知器运行时文件不存在于磁盘上，它们会被静默跳过。

## 通过 MCP 连接更多服务

除了工件交付流程外，智能体还可以通过 MCP（模型上下文协议）访问其他服务。MCP 生态系统为大多数流行工具提供了社区服务器——安装您需要的即可：

| 服务 | 解锁的功能 |
|---|---|
| **Notion** | 读/写 Notion 页面、数据库、查询工作区 |
| **GitHub** | Issues、PRs、评论、超越 gh CLI 的仓库搜索 |
| **Linear** | 工单、项目、周期 |
| **Slack** | 工作区范围搜索、读取其他频道 |
| **Gmail** | 收件箱整理、发送邮件、标签管理 |
| **Salesforce** | 线索、机会、账户数据 |
| **Snowflake / BigQuery** | 对数据仓库进行 SQL 查询 |
| **Google Drive** | 文件搜索、内容、共享管理 |

通过 `~/.hermes/config.yaml` 中的 `mcp_servers` 部分安装 MCP 服务器。有关完整设置指南，请参阅 [MCP 集成](./mcp.md)。

## 与 Perplexity Computer in Slack 的比较

Perplexity Computer 的 Slack 集成基于相同的理念构建：智能体生成可交付物（图表、PDF、幻灯片）并将其作为原生附件发回话题。Hermes 智能体的可交付模式在本地提供了相同的用户面向模式：

-   生成发生在用户自己的虚拟环境/沙箱中（无远程租户）。
-   文件通过相同的 Slack `files.uploadV2` API 进入聊天。
-   连接器的广度通过 MCP 实现，而不是通过一个精心策划的 400 个托管集成目录——只需安装您实际使用的那些。

OAuth 令牌保存在用户机器的 `auth.json` / `.env` 中。没有托管的令牌存储。没有多租户微虚拟机。最终结果相同。
