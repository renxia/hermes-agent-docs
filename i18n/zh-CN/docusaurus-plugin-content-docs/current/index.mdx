## 安装

### Windows 或 macOS

要轻松安装命令行和桌面应用程序，请从我们的网站下载 [Hermes 桌面安装程序](https://hermes-agent.nousresearch.com/) 并运行它。

### 无需 Hermes 桌面版：

对于仅限命令行、不使用 Hermes 桌面版的安装，请运行：

#### Linux / macOS / WSL2 / Android (Termux)

```bash
curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash
```

#### Windows（原生）

在 PowerShell 中运行：

```powershell
iex (irm https://hermes-agent.nousresearch.com/install.ps1)
```

请参阅完整的[安装指南](/getting-started/installation)，了解安装程序的功能、用户级与根级布局，以及 Windows 特定的注意事项。

:::tip 通往可用智能体的最快路径
安装后，运行 `hermes setup --portal` — 一个 OAuth 涵盖了一个模型和所有四个工具网关工具（网络搜索、图像生成、TTS、浏览器）。请参阅 [Nous Portal](/integrations/nous-portal)。
:::

## 什么是 Hermes 智能体？

它不是一个与 IDE 绑定的编码副驾驶，也不是围绕单个 API 的聊天机器人封装。它是一个**自主智能体**，运行时间越长，能力就越强。它存在于您所在之处——一台 5 美元的 VPS、一个 GPU 集群，或一种几乎不耗电的无服务器基础设施（Daytona, Modal）。您可以从 Telegram 与它对话，而它正在一台您从未通过 SSH 连接过的云虚拟机上工作。它不是绑定在您的笔记本电脑上的。

## 主要功能

*   **一个封闭的学习循环** — 智能体策展的记忆，定期进行引导，自主技能创建，使用过程中的技能自我改进，FTS5 跨会话召回和 LLM 摘要，以及 [Honcho](https://github.com/plastic-labs/honcho) 方言用户建模
*   **随处运行，不限于您的笔记本电脑** — 6 个终端后端：本地、Docker、SSH、Daytona、Singularity、Modal。Daytona 和 Modal 提供无服务器持久性——当空闲时，您的环境几乎不耗电。
*   **存在于您所在之处** — CLI、Telegram、Discord、Slack、WhatsApp、Signal、Matrix、Mattermost、Email、SMS、DingTalk、Feishu、WeCom、Weixin、QQ Bot、Yuanbao、BlueBubbles、Home Assistant、Microsoft Teams、Google Chat 等——来自一个网关的 20+ 个平台。
*   **由模型训练者构建** — 由 [Nous Research](https://nousresearch.com) 创建，它是 Hermes, Nomos 和 Psyche 的幕后实验室。可与 [Nous Portal](https://portal.nousresearch.com)、[OpenRouter](https://openrouter.ai)、OpenAI 或任何端点配合使用。
*   **计划自动化** — 内置 cron 定时任务，可投递到任何平台。
*   **委托和并行化** — 孵化隔离的子智能体以实现并行工作流。通过 `execute_code` 进行程序化工具调用，将多步骤流程合并为单次推理调用。
*   **开放标准技能** — 与 [agentskills.io](https://agentskills.io) 兼容。技能是可移植、可共享和社区贡献的，通过 Skills Hub 实现。
*   **完整的网络控制** — 搜索、提取、浏览、视觉、图像生成、TTS——一个 [Nous Portal](/integrations/nous-portal) 订阅即可捆绑所有功能。
*   **MCP 支持** — 连接到任何 MCP 服务器以获得扩展的工具能力。
*   **研究就绪** — 批量处理、轨迹导出、使用 Atropos 进行 RL 训练。由 [Nous Research](https://nousresearch.com) 构建——它是 Hermes, Nomos 和 Psyche 模型背后的实验室。

## 对于 LLM 和编码智能体

可供机器读取的文档入口：

*   **[`/llms.txt`](/llms.txt)** — 包含简短描述的每个文档页策展索引。约 17 KB，安全地加载到 LLM 上下文。
*   **[`/llms-full.txt`](/llms-full.txt)** — 将所有文档页面拼接成一个 markdown 文件供一次性摄取。约 1.8 MB。

这两个文件也解析为 `/docs/llms.txt` 和 `/docs/llms-full.txt`。每次部署时都会重新生成。
